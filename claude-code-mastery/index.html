1. First 5 Minutes

When you launch Claude Code, you're in a conversation. Type naturally.

$ claude
> fix the login bug in auth.py

Claude reads your project, understands the codebase, and acts. But how you prompt determines the quality.

Three things happen automatically:

1 Claude discovers and reads your CLAUDE.md (project rules)

2 Claude loads your memories from ~/.claude/memory/

3 Claude detects your git branch, recent changes, and environment

Your first commands to learn:

  • /help — see all commands
  • /model — switch between Opus/Sonnet/Haiku
  • /cost — see how much you've spent this session
  • /compact — compress conversation when context gets large
  • /clear — start fresh

2. Prompting Like a Pro

The Golden Rules

Rule 1: Be specific about WHAT, not HOW

❌ "Can you look at the auth module and maybe fix the issue with tokens?"
✅ "Fix the JWT refresh token bug in src/auth/tokens.py — tokens expire but aren't auto-refreshed"

Rule 2: Give context upfront

❌ "This is broken"
✅ "Running `pytest tests/test_auth.py` fails with 'TokenExpiredError' on line 42. The refresh endpoint returns 401."

Rule 3: One task per message

❌ "Fix the bug, add tests, update the README, and deploy"
✅ "Fix the JWT refresh bug in src/auth/tokens.py"
(then after it's done)
✅ "Now add tests for the fix"

Rule 4: Use file paths

Claude works faster when you point to exact locations:

✅ "Review src/api/handlers/user.py:45-80 — the validation logic looks wrong"
✅ "The error is in packages/core/src/index.ts"

Rule 5: State the outcome you want

✅ "Make the tests pass: npm test should show 0 failures"
✅ "After your fix, `curl localhost:3000/api/health` should return 200"

Power Prompting Patterns

The Debug Pattern:

Error: ConnectionRefusedError in src/db/pool.py:23
Stack trace: [paste it]
This started after I upgraded postgres to v16. Fix it.

The Review Pattern:

Review src/payments/stripe.py for security issues. Focus on input validation and error handling.

The Refactor Pattern:

Refactor src/utils/helpers.py — it's 800 lines. Split into focused modules. Keep all tests passing.

The Explain Pattern:

Walk me through how src/engine/query.rs works. I need to understand the message loop.

The Research Pattern:

What's the best way to implement rate limiting in FastAPI? Compare token bucket vs sliding window. Show me code.

5. CLAUDE.md — Your Project Brain

This is the #1 power-user feature. Create a CLAUDE.md in your project root and Claude reads it every session.

Where to put it

my-project/
├── CLAUDE.md              ← Project-wide rules
├── .claude/
│   ├── CLAUDE.md          ← Alternative location
│   └── rules/
│       ├── python.md      ← Language-specific rules
│       ├── testing.md     ← Testing conventions
│       └── deploy.md      ← Deploy procedures

What to put in it

# Project: MyApp

## Architecture
FastAPI backend + React frontend. PostgreSQL database.
API lives in src/api/, frontend in web/src/.

## Code Style
- Python: type hints everywhere, use ruff for formatting
- TypeScript: strict mode, prefer const, async/await
- No print statements in production code — use logging

## Testing
- Run: pytest tests/ -x (stop on first failure)
- Frontend: npm test --prefix web
- Always write tests before fixing bugs

## Git
- Branch naming: feat/, fix/, docs/
- Commit messages: imperative mood ("Add feature" not "Added feature")
- Never force-push to main

## Important
- The payments module (src/payments/) is critical — always run full test suite after changes
- Environment variables are in .env.example — never commit .env
- Database migrations: alembic upgrade head

Pro Tips

  • Be specific — "use ruff" beats "format your code"
  • Include commands — give Claude the exact test/build/lint commands
  • Mark critical areas — tell Claude which files need extra care
  • Keep it under 200 lines — too long wastes context tokens
  • Check it into git — your whole team benefits

6. Memory System

Claude remembers things between sessions. Four types:

Memory Types

TypeWhat to saveScope
**user**Your preferences, expertise, work stylePrivate
**feedback**"Always run tests before committing"Private or Team
**project**Current sprint goals, deadlines, blockersTeam
**reference**Links to Jira board, Slack channel, docsTeam

How it works

  • Auto-save: Claude detects important context and saves automatically
  • Manual save: Tell Claude "remember this" or use /memory
  • Storage: ~/.claude/memory/ (private), .claude/memory/ (team/shared)
  • Recall: Memories load automatically at session start

Pro Tips

"Remember: I prefer detailed explanations with code examples"
"Remember for this project: the API uses v2 endpoints at /api/v2/"
"Remember: always use pytest-asyncio for async test fixtures"

10. MCP Servers

MCP (Model Context Protocol) servers add new tools to Claude.

Add a Server

# From CLI
claude mcp add github-tools --config '{"command": "npx", "args": ["-y", "@anthropic/github-mcp"]}'

# In session
/mcp

Popular Servers

  • GitHub — PR reviews, issue management
  • Linear — Issue tracking
  • Slack — Read/send messages
  • Web Search — Real-time web access
  • Database — Direct SQL queries

Config File

~/.claude/.mcp.json:

{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@anthropic/github-mcp"],
      "env": { "GITHUB_TOKEN": "ghp_..." }
    }
  }
}

13. CLI Flags

Daily Use

claude                              # Start new session
claude -c                           # Continue last session
claude -r <session-id>              # Resume specific session
claude -p "fix the bug in auth.py"  # Non-interactive (for scripts)
claude --model opus --effort high   # Smart mode

Advanced

claude --bare --system-prompt "You are a Python expert"  # Minimal mode
claude --add-dir ../shared-lib      # Include extra directories
claude --max-turns 10               # Limit agent loops
claude --max-budget-usd 5.00        # Spending cap
claude --output-format json         # Machine-readable output
claude --worktree feature-x         # Isolated git worktree

Create .claude/hooks/ with TypeScript files for custom automation:

  • setup.ts — runs on project init
  • sessionStart.ts — runs on every session
  • beforeChat.ts — runs before each message

Session Tricks

  • Fork sessions: --fork-session when resuming to branch conversation
  • Export: /export to save conversation as markdown
  • Copy: /copy to clipboard
  • Compact early: Run /compact proactively at ~50% context usage

15. Pro Workflows

Workflow 1: Bug Fix (5 minutes)

1. claude --permission-mode acceptEdits
2. "The login endpoint returns 500 when email has uppercase. Error in src/api/auth.py"
3. (Claude finds and fixes the bug)
4. "Run the tests"
5. "Commit with message: Fix case-sensitive email validation"

Workflow 2: Feature Implementation (30 minutes)

1. claude --model opus --effort high
2. "I need to add rate limiting to the API. Read src/api/ and suggest an approach."
3. "/plan"  (enter plan mode)
4. "Implement the plan. Use token bucket algorithm. Add tests."
5. "/diff"  (review changes)
6. "Run all tests and fix any failures"

Workflow 3: Code Review

1. claude
2. "Review the changes in the last 3 commits. Focus on security and performance."
3. (Claude reads git log, diffs, and provides review)

Workflow 4: Codebase Exploration

1. claude --model haiku  (cheap exploration)
2. "How does the payment system work? Trace from checkout to Stripe."
3. "What files handle user authentication?"
4. "Show me the database schema for orders"

Workflow 5: Multi-Agent Pipeline

1. claude --model sonnet
2. "Use an explore agent to map all API endpoints. Then create a test plan."
3. (Claude spawns explore agent, gets results, creates plan)
4. "Execute the plan — implement missing tests"
5. "Spawn a verify agent to run the full test suite"
