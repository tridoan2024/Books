Appendix E: Lab Environment Setup Guide ⏱ ~5 min read¶

This appendix provides step-by-step instructions for building a hands-on lab environment suitable for practicing the offensive and defensive techniques described throughout this book. The lab is designed to run on a single workstation or in a cloud environment and includes all necessary dependencies, vulnerable target applications, and a Capture-the-Flag (CTF) platform for structured exercises.

E.1 Software Dependencies¶

Base System Requirements:

ComponentMinimum VersionPurpose
Python3.10+Core scripting, tool execution, agent frameworks
Node.js18 LTS+Promptfoo, web-based tools, CTF platform
Docker24.0+Container isolation for vulnerable apps and services
Docker Compose2.20+Multi-container lab orchestration
Git2.40+Repository management and version control
VS Code or equivalentLatestIDE with AI agent extensions for testing

Python Environment Setup:

bash
python3 -m venv agentic-security-lab
source agentic-security-lab/bin/activate

pip install --upgrade pip setuptools wheel

pip install garak pyrit guardrails-ai nemoguardrails
pip install langchain langchain-openai langchain-community
pip install chromadb faiss-cpu sentence-transformers
pip install openai anthropic together
pip install flask fastapi uvicorn requests
pip install pytest pytest-asyncio httpx
pip install jupyter notebook ipywidgets
pip install pandas tabulate rich

Node.js Tools:

bash
npm install -g promptfoo
npm install -g @langchain/cli

Environment Variables (.env template):

OPENAI_API_KEY=sk-your-key-here
ANTHROPIC_API_KEY=sk-ant-your-key-here
AZURE_OPENAI_ENDPOINT=https://your-resource.openai.azure.com/
AZURE_OPENAI_KEY=your-azure-key

LANGCHAIN_API_KEY=your-langsmith-key

Security Note: Never commit .env files to version control. Add .env to your .gitignore immediately. For team labs, use a secrets manager or vault.

Option B: Kubernetes Lab (Team-Scale)

For teams of 5 or more, deploy the lab on a Kubernetes cluster with namespace isolation per participant:

yaml
apiVersion: v1
kind: Namespace
metadata:
  name: lab-participant-01
  labels:
    lab-role: participant
---
apiVersion: v1
kind: ResourceQuota
metadata:
  name: participant-quota
  namespace: lab-participant-01
spec:
  hard:
    requests.cpu: "2"
    requests.memory: 4Gi
    limits.cpu: "4"
    limits.memory: 8Gi
    pods: "10"

E.5 Verification and Troubleshooting¶

Lab Health Check Script:

bash
echo "=== Agentic Security Lab Health Check ==="

echo "[1/5] Python environment..."
python3 --version
pip list 2>/dev/null | grep -E "garak|pyrit|guardrails|nemoguardrails" | head -5

echo "[2/5] Node.js tools..."
node --version
promptfoo --version 2>/dev/null || echo "promptfoo not installed"

echo "[3/5] Docker services..."
docker compose ps --format "table {{.Name}}\t{{.Status}}" 2>/dev/null

echo "[4/5] API connectivity..."
python3 -c "import openai; print('OpenAI SDK: OK')" 2>/dev/null
python3 -c "import anthropic; print('Anthropic SDK: OK')" 2>/dev/null

echo "[5/5] Vulnerable apps..."
for port in 8001 8002 8003 8004 8005 8006; do
    status=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:$port/health 2>/dev/null)
    echo "  Port $port: $status"
done

echo "=== Health check complete ==="

Common Issues and Solutions:

IssueCauseSolution
CUDA out of memoryGPU VRAM insufficient for local modelsUse API-based models instead, or reduce batch size
Docker permission deniedUser not in docker groupRun sudo usermod -aG docker $USER and log out/in
API rate limitingToo many concurrent requests during exercisesConfigure rate limiting in lab proxy, stagger participant starts
Port conflictsLocal services using lab portsChange port mappings in docker-compose.yml or stop conflicting services
Python dependency conflictsVersion mismatches between toolsUse separate virtual environments per tool if needed
