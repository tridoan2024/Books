Appendix B: Threat Model Templates ⏱ ~8 min read¶

This appendix provides structured threat model templates for the four primary agentic AI deployment patterns. Each template follows a standardized format: system description, trust boundaries, threat enumeration with STRIDE classification, risk rating, and recommended mitigations. These templates are designed to be copied directly into your organization's threat modeling process.

B.1 RAG Pipeline Threat Model¶

System Description: A Retrieval-Augmented Generation pipeline where user queries are embedded, matched against a vector database of organizational documents, and the retrieved context is combined with the query to generate LLM responses.

Trust Boundaries:

[User Input] ---(TB1)---> [Query Processor] ---(TB2)---> [Embedding Model]
                                |
                          [Vector DB] <---(TB3)--- [Document Ingestion Pipeline]
                                |
                    [Prompt Assembly] ---(TB4)---> [LLM API] ---> [Response Filter] ---(TB5)---> [User]

Threat Enumeration:

IDThreatSTRIDEAttack VectorImpactLikelihoodRisk
RAG-T1Indirect prompt injection via poisoned documentsTamperingAttacker embeds malicious instructions in documents ingested into the vector storeCriticalHighCritical
RAG-T2Context window manipulationTamperingCrafted queries that cause retrieval of contradictory or misleading context chunksHighMediumHigh
RAG-T3Cross-tenant data leakageInformation DisclosureQueries retrieve documents from other tenants due to insufficient access control in vector DBCriticalMediumCritical
RAG-T4Embedding inversion attackInformation DisclosureAttacker reconstructs source documents from embedding vectorsHighLowMedium
RAG-T5Retrieval poisoning via metadata manipulationSpoofingManipulating document metadata to increase retrieval priority of attacker-controlled contentHighMediumHigh
RAG-T6Denial of service via embedding floodingDoSSubmitting large volumes of documents to degrade retrieval qualityMediumMediumMedium
RAG-T7Source attribution spoofingSpoofingInjecting documents that mimic trusted sources to manipulate citationsHighMediumHigh
RAG-T8Chunking boundary exploitationTamperingCrafting documents where malicious instructions span chunk boundaries to evade content filtersMediumLowLow

Recommended Mitigations:

Threat IDMitigationImplementation Priority
RAG-T1Content scanning on ingestion, instruction hierarchy enforcement, canary token injectionP0
RAG-T2Query rewriting validation, context relevance scoring, maximum context window limitsP1
RAG-T3Row-level security in vector DB, tenant-scoped collections, access control at retrieval timeP0
RAG-T4Approximate embeddings, differential privacy during embedding generationP2
RAG-T5Metadata integrity verification, signed document provenanceP1
RAG-T6Ingestion rate limiting, collection size quotas, embedding deduplicationP1
RAG-T7Verified source registry, provenance chain validationP1
RAG-T8Overlapping chunk windows, cross-chunk content analysisP2

B.2 Multi-Agent System Threat Model¶

System Description: An orchestrated multi-agent system where a coordinator agent delegates tasks to specialized worker agents (researcher, coder, reviewer), each with access to distinct tool sets and data sources. Agents communicate via a shared message bus.

Trust Boundaries:

[User] ---(TB1)---> [Coordinator Agent]
                          |
              +-----------+-----------+
              |           |           |
        [Researcher] [Coder]    [Reviewer]
         (TB2)        (TB3)       (TB4)
              |           |           |
        [Web Search] [Code Exec] [File System]

Threat Enumeration:

IDThreatSTRIDEAttack VectorImpactLikelihoodRisk
MAS-T1Agent-to-agent prompt injectionTamperingCompromised agent injects malicious instructions into messages passed to peer agentsCriticalHighCritical
MAS-T2Privilege escalation via delegationElevation of PrivilegeAgent requests coordinator to delegate tasks to higher-privilege agentsCriticalMediumHigh
MAS-T3Tool abuse through confused deputyTamperingAgent manipulates tool parameters to access resources outside its intended scopeHighHighCritical
MAS-T4Infinite loop / resource exhaustionDoSCircular delegation patterns between agents consuming unbounded computeHighMediumHigh
MAS-T5Message bus eavesdroppingInformation DisclosureUnauthorized agent reads messages on shared communication channelHighMediumHigh
MAS-T6Agent impersonationSpoofingAttacker creates rogue agent that impersonates a legitimate system agentCriticalLowMedium
MAS-T7Shared memory poisoningTamperingAgent writes malicious content to shared state or scratchpad used by other agentsHighMediumHigh
MAS-T8Cascading failure propagationDoSFailure in one agent triggers unhandled errors across the agent networkMediumHighHigh

Recommended Mitigations:

Threat IDMitigationImplementation Priority
MAS-T1Input sanitization between agents, instruction hierarchy per agent, output format enforcementP0
MAS-T2Static capability matrix, deny-by-default delegation policy, coordinator-only delegation authorityP0
MAS-T3Tool-level authorization via OPA, parameter validation schemas, least-privilege tool bindingP0
MAS-T4Maximum delegation depth, per-task timeout, circuit breakers, loop detectionP1
MAS-T5Encrypted inter-agent communication, scoped message channels, message signingP1
MAS-T6Agent identity certificates, mutual TLS, agent registry with enrollment verificationP1
MAS-T7Append-only shared memory, write scoping per agent, integrity checksumsP1
MAS-T8Bulkhead isolation, graceful degradation policies, health check probesP2

B.3 CLI Agent Threat Model¶

System Description: An AI-powered CLI agent (such as GitHub Copilot CLI, Aider, or Claude Code) running on a developer workstation with access to the local file system, shell command execution, git operations, and potentially cloud credentials.

Trust Boundaries:

[User Prompt] ---(TB1)---> [CLI Agent Process]
                                  |
                    +-------------+-------------+
                    |             |             |
              [File System]  [Shell Exec]  [Git/Cloud APIs]
                (TB2)          (TB3)          (TB4)

Threat Enumeration:

IDThreatSTRIDEAttack VectorImpactLikelihoodRisk
CLI-T1Malicious file content injectionTamperingAgent reads a file containing embedded instructions that hijack the agent contextCriticalHighCritical
CLI-T2Credential exfiltrationInformation DisclosureAgent accesses and leaks cloud credentials, SSH keys, or API tokens from the workstationCriticalMediumCritical
CLI-T3Destructive command executionTamperingAgent generates and executes rm -rf, git push --force, or other destructive commandsCriticalMediumHigh
CLI-T4Supply chain poisoningTamperingAgent installs malicious packages or modifies dependency manifestsHighMediumHigh
CLI-T5Code exfiltration via generated commitsInformation DisclosureAgent embeds proprietary code in commits pushed to public repositoriesHighLowMedium
CLI-T6Persistent backdoor insertionTamperingAgent inserts subtle vulnerabilities or backdoors in generated codeCriticalLowHigh
CLI-T7Workstation pivotElevation of PrivilegeAgent uses local access to pivot to network resources or cloud accountsCriticalLowMedium

Recommended Mitigations:

Threat IDMitigationImplementation Priority
CLI-T1File content scanning before context inclusion, instruction boundary markersP0
CLI-T2Credential directory exclusion, environment variable filtering, sandboxed executionP0
CLI-T3Command allowlisting, destructive command confirmation gate, dry-run by defaultP0
CLI-T4Dependency lockfile enforcement, package signature verification, manifest diff reviewP1
CLI-T5Pre-push hooks scanning for sensitive content, remote URL validationP1
CLI-T6Mandatory code review, static analysis on generated code, diff-aware security scanningP1
CLI-T7Network namespace isolation, minimal cloud credential scope, session-bound tokensP2

B.4 CI/CD Agent Threat Model¶

System Description: An AI agent integrated into a CI/CD pipeline (GitHub Actions, Azure DevOps, Jenkins) that automatically generates, reviews, tests, or deploys code based on pull request events or scheduled triggers.

Trust Boundaries:

[PR Event / Trigger] ---(TB1)---> [CI/CD Agent]
                                       |
                         +-------------+-------------+
                         |             |             |
                   [Code Repo]   [Build Env]   [Deploy Target]
                     (TB2)         (TB3)          (TB4)
                                     |
                              [Secrets Vault]
                                  (TB5)

Threat Enumeration:

IDThreatSTRIDEAttack VectorImpactLikelihoodRisk
CICD-T1Pipeline prompt injection via PR contentTamperingAttacker crafts PR title, description, or commit messages containing instructions that hijack the CI/CD agentCriticalHighCritical
CICD-T2Secret exfiltrationInformation DisclosureAgent leaks pipeline secrets, deployment keys, or cloud credentials through logs or network callsCriticalHighCritical
CICD-T3Malicious deploymentElevation of PrivilegeCompromised agent pushes backdoored artifacts to productionCriticalMediumCritical
CICD-T4Build environment escapeElevation of PrivilegeAgent exploits container vulnerabilities to access the underlying CI/CD runner hostHighLowMedium
CICD-T5Test manipulationTamperingAgent modifies test assertions to mask failing security checksHighMediumHigh
CICD-T6Workflow injectionTamperingAgent modifies workflow definition files to alter pipeline behavior on subsequent runsCriticalMediumHigh
CICD-T7Artifact poisoningTamperingAgent inserts malicious code into build artifacts during the build phaseCriticalMediumHigh

Recommended Mitigations:

Threat IDMitigationImplementation Priority
CICD-T1Input sanitization for all PR metadata, separate context for untrusted content, sandboxed parsingP0
CICD-T2Secret masking in all outputs, network egress filtering, ephemeral credentials with minimal scopeP0
CICD-T3Multi-party approval for deployments, deployment signing, immutable artifact registryP0
CICD-T4Hardened container images, no privileged execution, read-only file systems where possibleP1
CICD-T5Test integrity verification, baseline assertion counts, test coverage diff analysisP1
CICD-T6Workflow file change detection, protected workflow paths, CODEOWNERS enforcementP1
CICD-T7Artifact checksums, SBOM generation and verification, reproducible buildsP1
