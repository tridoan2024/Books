Appendix D: Regulatory and Compliance Mapping ⏱ ~6 min read¶

This appendix maps agentic AI security controls to major regulatory frameworks and compliance standards. Use these mappings to demonstrate compliance coverage, identify gaps, and align your AI security program with organizational governance requirements.

D.1 EU AI Act Compliance Mapping¶

The EU AI Act (Regulation 2024/1689) establishes a risk-based regulatory framework for AI systems. Agentic AI systems that make autonomous decisions, interact with users, or process personal data are likely classified as high-risk under Title III.

Technical documentationArt. 11Architecture diagrams, system prompt documentation, tool capability manifests, agent interaction flowsTechnical design docs, API specifications, capability matrices
Record keeping and loggingArt. 12Full conversation audit trails, agent action logs, tool invocation records, decision tracesLog retention policies, SIEM integration evidence, trace samples
Transparency and user informationArt. 13AI disclosure to end users, explanation of agent capabilities and limitations, confidence indicatorsUser-facing documentation, UI disclosure elements, transparency reports

Accuracy, robustness, cybersecurityArt. 15Input validation, output filtering, adversarial testing, guardrails, monitoring and alertingTest results, guardrail configurations, incident response records
Quality management systemArt. 17Security development lifecycle, change management, version control, continuous monitoringQMS documentation, CI/CD security gates, change approval records
Conformity assessmentArt. 43Internal or third-party assessment of AI system complianceAssessment reports, certification documentation
Post-market monitoringArt. 72Production monitoring, drift detection, incident reporting, model performance trackingMonitoring dashboards, incident reports, performance baselines

Key Obligations for Agentic AI Deployers:

  • Register high-risk AI systems in the EU database before deployment
  • Conduct a Fundamental Rights Impact Assessment (FRIA) for public-sector deployments
  • Implement automated incident detection and reporting to national authorities within 72 hours
  • Maintain human oversight capability proportional to the risk level of agent autonomous actions

Audit Evidence Checklist for SOC 2 / ISO 27001:

  • Agent capability manifests and tool authorization matrices
  • Prompt template change history with approval records
  • Red team test reports and vulnerability remediation tracking
  • Agent monitoring dashboard screenshots and alerting configurations
  • Incident response records for any AI-related security events
  • Third-party LLM provider risk assessments and SLA compliance reports
  • Data flow diagrams showing agent data handling and storage locations
