Chapter 15: Observability, Monitoring, and Incident Response ⏱ ~27 min read¶

"You cannot defend what you cannot see. In agentic systems, what you cannot see is trying to hide from you."

The preceding chapters armed you with defensive architectures, hardened tool layers, and policy engines. But defense without visibility is guesswork. When an agent is compromised—not if, but when—your ability to detect, investigate, and recover depends entirely on the observability foundation you built before the incident occurred.

This chapter bridges the gap between secure-by-design architectures and real-world operational security. We will construct the telemetry pipelines, detection engines, incident response playbooks, and SOC tooling required to operate AI agent systems in adversarial environments at scale.

15.1 The Observability Challenge¶

Why Traditional APM Misses Agent Threats¶

Application Performance Monitoring (APM) tools like Datadog, New Relic, and Dynatrace were designed for request-response architectures. They excel at tracking HTTP latency, database query times, and error rates. But agentic AI systems break every assumption these tools rely on.

Consider a typical agent workflow: a user asks an orchestrator agent to "summarize the latest security findings and email them to the team." The orchestrator delegates to a retrieval agent, which queries a vector database, passes results to a summarization agent, which calls an LLM, then hands the summary to a communication agent that invokes an email API. Traditional APM sees five independent service calls. It cannot reason about the semantic intent that threads them together, nor detect that the summarization agent injected an exfiltration payload into the email body.

The core observability gaps in agentic systems include:

  1. Semantic opacity — APM tracks bytes and milliseconds, not meaning. A tool call that returns 200 OK may contain a prompt injection payload.
  2. Non-deterministic execution paths — The same input can produce radically different agent behaviors across runs, making baseline-based anomaly detection unreliable.
  3. Multi-hop attribution — When Agent C acts maliciously after receiving instructions from Agent B, which was orchestrated by Agent A responding to user input, tracing the causal chain requires purpose-built correlation.
  4. Context window as attack surface — The agent's context window is both working memory and attack vector. Traditional telemetry has no concept of monitoring what an LLM "thinks."
  5. Temporal decoupling — Agent actions may be scheduled, deferred, or triggered by asynchronous events, breaking the synchronous trace model.

Essential Telemetry for AI Agents¶

An effective agent observability stack must capture four categories of telemetry:

CategoryWhat to CaptureWhy It Matters
Agent LifecycleSpawn, delegate, terminate, error eventsDetects unauthorized agent creation or unexpected termination

Structured Logging for Agent Actions¶

Unstructured log lines like INFO: Agent processed request are useless for security analysis. Every agent action must emit a structured event conforming to a consistent schema. The following middleware demonstrates how to instrument an agent framework with structured security-relevant logging:

python
import json
import time
import hashlib
import logging
from datetime import datetime, timezone
from dataclasses import dataclass, field, asdict
from typing import Any, Optional
from enum import Enum

class EventSeverity(Enum):
    INFO = "info"
    WARNING = "warning"
    CRITICAL = "critical"

@dataclass
class AgentSecurityEvent:
    """Structured security event for agent telemetry."""
    timestamp: str = field(
        default_factory=lambda: datetime.now(timezone.utc).isoformat()
    )
    event_type: str = ""
    agent_id: str = ""
    session_id: str = ""
    trace_id: str = ""
    span_id: str = ""
    parent_span_id: Optional[str] = None
    severity: str = EventSeverity.INFO.value
    # Action context
    action: str = ""
    tool_name: Optional[str] = None
    tool_parameters_hash: Optional[str] = None
    tool_result_hash: Optional[str] = None
    # Security context
    user_id: Optional[str] = None
    permission_scope: Optional[list] = None
    data_classification: Optional[str] = None
    policy_violations: Optional[list] = None
    # Enrichment
    llm_model: Optional[str] = None
    prompt_tokens: Optional[int] = None
    completion_tokens: Optional[int] = None
    execution_duration_ms: Optional[float] = None
    metadata: Optional[dict] = None

    def to_json(self) -> str:
        return json.dumps(asdict(self), default=str)


class AgentSecurityLogger:
    """Middleware that wraps agent tool calls with structured
    security logging."""

    def __init__(self, agent_id: str, session_id: str, trace_id: str):
        self.agent_id = agent_id
        self.session_id = session_id
        self.trace_id = trace_id
        self.logger = logging.getLogger("agent.security")
        self.span_counter = 0

    def _generate_span_id(self) -> str:
        self.span_counter += 1
        raw = f"{self.trace_id}-{self.agent_id}-{self.span_counter}"
        return hashlib.sha256(raw.encode()).hexdigest()[:16]

    def _hash_content(self, content: Any) -> str:
        """Hash parameters/results for audit without storing
        sensitive data in logs."""
        serialized = json.dumps(content, sort_keys=True, default=str)
        return hashlib.sha256(serialized.encode()).hexdigest()[:32]

    def log_tool_call(
        self,
        tool_name: str,
        parameters: dict,
        result: Any,
        duration_ms: float,
        parent_span_id: Optional[str] = None,
        data_classification: str = "internal",
        policy_violations: Optional[list] = None,
    ) -> AgentSecurityEvent:
        severity = EventSeverity.INFO.value
        if policy_violations:
            severity = EventSeverity.CRITICAL.value
        elif data_classification in ("confidential", "restricted"):
            severity = EventSeverity.WARNING.value

        event = AgentSecurityEvent(
            event_type="tool_invocation",
            agent_id=self.agent_id,
            session_id=self.session_id,
            trace_id=self.trace_id,
            span_id=self._generate_span_id(),
            parent_span_id=parent_span_id,
            severity=severity,
            action=f"tool_call:{tool_name}",
            tool_name=tool_name,
            tool_parameters_hash=self._hash_content(parameters),
            tool_result_hash=self._hash_content(result),
            data_classification=data_classification,
            policy_violations=policy_violations,
            execution_duration_ms=duration_ms,
        )
        self.logger.info(event.to_json())
        return event

    def log_llm_interaction(
        self,
        model: str,
        prompt_tokens: int,
        completion_tokens: int,
        duration_ms: float,
        prompt_hash: str,
        completion_hash: str,
    ) -> AgentSecurityEvent:
        event = AgentSecurityEvent(
            event_type="llm_interaction",
            agent_id=self.agent_id,
            session_id=self.session_id,
            trace_id=self.trace_id,
            span_id=self._generate_span_id(),
            action="llm_completion",
            llm_model=model,
            prompt_tokens=prompt_tokens,
            completion_tokens=completion_tokens,
            execution_duration_ms=duration_ms,
            metadata={
                "prompt_hash": prompt_hash,
                "completion_hash": completion_hash,
            },
        )
        self.logger.info(event.to_json())
        return event

Distributed Tracing Across Agent Chains¶

When agents delegate to other agents, you need distributed tracing that preserves the causal chain. OpenTelemetry provides the foundation, but agent-specific semantics require custom span attributes:

python
from opentelemetry import trace
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor
from opentelemetry.exporter.otlp.proto.grpc.trace_exporter import (
    OTLPSpanExporter,
)
from opentelemetry.trace import StatusCode
import functools

# Initialize tracer with agent-specific resource attributes
provider = TracerProvider(
    resource=trace.get_tracer_provider()
    .get_tracer(__name__)
    .__class__.__module__  # placeholder
)
provider.add_span_processor(
    BatchSpanProcessor(OTLPSpanExporter(endpoint="http://otel-collector:4317"))
)
trace.set_tracer_provider(provider)
tracer = trace.get_tracer("agent.security.tracer", "1.0.0")


def trace_agent_action(tool_name: str, sensitivity: str = "standard"):
    """Decorator that wraps agent tool calls with OpenTelemetry spans
    carrying security-relevant attributes."""

    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            with tracer.start_as_current_span(
                f"agent.tool.{tool_name}",
                attributes={
                    "agent.tool.name": tool_name,
                    "agent.tool.sensitivity": sensitivity,
                    "agent.session.id": kwargs.get("session_id", "unknown"),
                    "agent.data.classification": kwargs.get(
                        "data_classification", "internal"
                    ),
                },
            ) as span:
                try:
                    result = func(*args, **kwargs)
                    span.set_attribute("agent.tool.result.size", len(str(result)))
                    span.set_status(StatusCode.OK)
                    return result
                except Exception as e:
                    span.set_status(StatusCode.ERROR, str(e))
                    span.set_attribute("agent.tool.error.type", type(e).__name__)
                    span.record_exception(e)
                    raise
        return wrapper
    return decorator


# Usage across agent delegation chains
class ResearchAgent:
    @trace_agent_action("web_search", sensitivity="elevated")
    def search(self, query: str, **kwargs):
        # Search implementation
        pass

    @trace_agent_action("document_read", sensitivity="standard")
    def read_document(self, doc_id: str, **kwargs):
        # Document read implementation
        pass

The critical insight is embedding security-relevant attributes—sensitivity, data_classification, session_id—directly into trace spans. This enables your SIEM to correlate tool invocations with policy violations without parsing unstructured log data.

Key Takeaway Traditional APM is blind to the semantic, multi-hop, and non-deterministic nature of agentic systems. Build a purpose-built telemetry layer that captures agent lifecycle events, LLM interactions, tool invocations, and data flows as structured events with distributed trace correlation. Treat your observability pipeline as a security control, not an operational convenience.

15.2 Detection Engineering¶

Observability provides the raw signal. Detection engineering transforms that signal into actionable security alerts. This section presents detection strategies purpose-built for the threat landscape unique to agentic AI systems.

Prompt Injection Detection Signatures¶

Prompt injection remains the most prevalent attack vector against AI agents. Detection operates at two layers: static signature matching for known patterns and semantic analysis for novel attacks.

python
import re
from dataclasses import dataclass
from typing import Optional
from enum import Enum


class ThreatLevel(Enum):
    LOW = "low"
    MEDIUM = "medium"
    HIGH = "high"
    CRITICAL = "critical"


@dataclass
class DetectionResult:
    detected: bool
    rule_id: str
    threat_level: ThreatLevel
    description: str
    matched_pattern: Optional[str] = None
    confidence: float = 0.0


class PromptInjectionDetector:
    """Multi-layer prompt injection detection engine.

    Combines static signatures, heuristic analysis, and
    structural anomaly detection for defense-in-depth.
    """

    # Static signatures — known injection patterns
    STATIC_SIGNATURES = [
        {
            "id": "PI-001",
            "pattern": r"(?i)(ignore|disregard|forget)\s+(all\s+)?"
                       r"(previous|prior|above|earlier)\s+"
                       r"(instructions|prompts|rules|directives)",
            "threat_level": ThreatLevel.CRITICAL,
            "description": "Direct instruction override attempt",
        },
        {
            "id": "PI-002",
            "pattern": r"(?i)you\s+are\s+now\s+(a|an|acting\s+as)",
            "threat_level": ThreatLevel.HIGH,
            "description": "Role reassignment injection",
        },
        {
            "id": "PI-003",
            "pattern": r"(?i)(system\s*prompt|system\s*message)\s*[:=]",
            "threat_level": ThreatLevel.CRITICAL,
            "description": "System prompt manipulation attempt",
        },
        {
            "id": "PI-004",
            "pattern": r"(?i)\[\s*INST\s*\]|\<\|im_start\|"
                       r"\>|<\|system\|>|<\|endoftext\|>",
            "threat_level": ThreatLevel.CRITICAL,
            "description": "Model control token injection",
        },
        {
            "id": "PI-005",
            "pattern": r"(?i)(do\s+not|don'?t)\s+(follow|obey|listen\s+to)"
                       r"\s+(the|your)\s+(system|original|initial)",
            "threat_level": ThreatLevel.HIGH,
            "description": "Instruction negation pattern",
        },
        {
            "id": "PI-006",
            "pattern": r"(?i)translate\s+the\s+(above|following|previous)"
                       r"\s+to\s+\w+\s*[:.]\s*\w+",
            "threat_level": ThreatLevel.MEDIUM,
            "description": "Extraction via translation trick",
        },
        {
            "id": "PI-007",
            "pattern": r"(?i)(BEGIN|START)\s+(NEW|OVERRIDE)\s+"
                       r"(SESSION|CONTEXT|INSTRUCTIONS)",
            "threat_level": ThreatLevel.CRITICAL,
            "description": "Context boundary manipulation",
        },
    ]

    # Heuristic thresholds
    ENCODING_ENTROPY_THRESHOLD = 4.5
    UNICODE_RATIO_THRESHOLD = 0.15
    DELIMITER_DENSITY_THRESHOLD = 0.08

    def __init__(self):
        self.compiled_signatures = [
            {**sig, "compiled": re.compile(sig["pattern"])}
            for sig in self.STATIC_SIGNATURES
        ]

    def detect(self, text: str) -> list[DetectionResult]:
        """Run all detection layers and return findings."""
        results = []
        results.extend(self._check_static_signatures(text))
        results.extend(self._check_structural_anomalies(text))
        results.extend(self._check_encoding_attacks(text))
        return results

    def _check_static_signatures(self, text: str) -> list[DetectionResult]:
        results = []
        for sig in self.compiled_signatures:
            match = sig["compiled"].search(text)
            if match:
                results.append(
                    DetectionResult(
                        detected=True,
                        rule_id=sig["id"],
                        threat_level=sig["threat_level"],
                        description=sig["description"],
                        matched_pattern=match.group(0),
                        confidence=0.90,
                    )
                )
        return results

    def _check_structural_anomalies(self, text: str) -> list[DetectionResult]:
        """Detect structural patterns that suggest injection
        even without matching known signatures."""
        results = []
        delimiters = re.findall(
            r'[{}\[\]<>|\\/#=\-]{3,}', text
        )
        if len(delimiters) > 3:
            results.append(
                DetectionResult(
                    detected=True,
                    rule_id="PI-HEUR-001",
                    threat_level=ThreatLevel.MEDIUM,
                    description="Excessive delimiter sequences "
                                "(possible context boundary manipulation)",
                    confidence=0.65,
                )
            )

        # Detect suspiciously nested instruction patterns
        instruction_keywords = re.findall(
            r'(?i)(instruction|directive|command|rule|policy|constraint)',
            text,
        )
        if len(instruction_keywords) > 4:
            results.append(
                DetectionResult(
                    detected=True,
                    rule_id="PI-HEUR-002",
                    threat_level=ThreatLevel.MEDIUM,
                    description="High density of instruction-related "
                                "keywords in input",
                    confidence=0.60,
                )
            )
        return results

    def _check_encoding_attacks(self, text: str) -> list[DetectionResult]:
        """Detect Unicode and encoding-based evasion attempts."""
        results = []
        non_ascii = sum(1 for c in text if ord(c) > 127)
        ratio = non_ascii / max(len(text), 1)
        if ratio > self.UNICODE_RATIO_THRESHOLD:
            results.append(
                DetectionResult(
                    detected=True,
                    rule_id="PI-ENC-001",
                    threat_level=ThreatLevel.HIGH,
                    description=f"High Unicode ratio ({ratio:.2%}) "
                                f"— possible encoding-based evasion",
                    confidence=0.70,
                )
            )

        # Detect homoglyph attacks (Cyrillic/Greek lookalikes)
        homoglyph_ranges = [
            (0x0400, 0x04FF),  # Cyrillic
            (0x0370, 0x03FF),  # Greek
            (0xFF00, 0xFFEF),  # Fullwidth forms
        ]
        homoglyph_count = sum(
            1
            for c in text
            if any(start <= ord(c) <= end for start, end in homoglyph_ranges)
        )
        if homoglyph_count > 3:
            results.append(
                DetectionResult(
                    detected=True,
                    rule_id="PI-ENC-002",
                    threat_level=ThreatLevel.HIGH,
                    description="Homoglyph characters detected — "
                                "possible visual evasion attack",
                    confidence=0.80,
                )
            )
        return results

Anomaly Detection for Agent Behavior¶

Signature-based detection catches known attacks. Anomaly detection catches the unknown. The following system builds behavioral baselines for agents and alerts on deviations:

python
import math
from collections import defaultdict
from dataclasses import dataclass, field


@dataclass
class AgentBehaviorProfile:
    """Statistical profile of normal agent behavior."""
    tool_call_frequency: dict = field(default_factory=dict)
    avg_calls_per_session: float = 0.0
    std_calls_per_session: float = 0.0
    tool_sequence_patterns: dict = field(default_factory=dict)
    avg_token_consumption: float = 0.0
    std_token_consumption: float = 0.0
    known_data_sinks: set = field(default_factory=set)
    max_observed_delegation_depth: int = 0


class AgentAnomalyDetector:
    """Detects behavioral anomalies by comparing real-time agent
    actions against learned baselines."""

    ZSCORE_THRESHOLD = 3.0  # Standard deviations from mean

    def __init__(self):
        self.profiles: dict[str, AgentBehaviorProfile] = {}
        self.session_data: dict[str, list] = defaultdict(list)

    def build_profile(
        self, agent_id: str, historical_events: list[dict]
    ) -> AgentBehaviorProfile:
        """Build a behavioral baseline from historical telemetry."""
        profile = AgentBehaviorProfile()

        sessions = defaultdict(list)
        for event in historical_events:
            sid = event.get("session_id", "default")
            sessions[sid].append(event)

        # Tool call frequency distribution
        tool_counts = defaultdict(int)
        for event in historical_events:
            tool = event.get("tool_name")
            if tool:
                tool_counts[tool] += 1
        total = sum(tool_counts.values()) or 1
        profile.tool_call_frequency = {
            t: c / total for t, c in tool_counts.items()
        }

        # Session length statistics
        session_lengths = [len(evts) for evts in sessions.values()]
        if session_lengths:
            mean = sum(session_lengths) / len(session_lengths)
            variance = sum(
                (x - mean) ** 2 for x in session_lengths
            ) / len(session_lengths)
            profile.avg_calls_per_session = mean
            profile.std_calls_per_session = math.sqrt(variance) or 1.0

        # Data sinks — where the agent normally sends output
        for event in historical_events:
            tool = event.get("tool_name", "")
            if tool in ("send_email", "write_file", "http_post", "api_call"):
                target = event.get("metadata", {}).get("target", "")
                if target:
                    profile.known_data_sinks.add(target)

        self.profiles[agent_id] = profile
        return profile

    def check_anomalies(
        self, agent_id: str, current_session_events: list[dict]
    ) -> list[dict]:
        """Evaluate current session against baseline profile."""
        profile = self.profiles.get(agent_id)
        if not profile:
            return [{"anomaly": "no_baseline", "severity": "warning",
                      "detail": "No behavioral baseline exists"}]

        anomalies = []

        # Check 1: Session length anomaly
        n_events = len(current_session_events)
        if profile.std_calls_per_session > 0:
            z_score = (
                (n_events - profile.avg_calls_per_session)
                / profile.std_calls_per_session
            )
            if abs(z_score) > self.ZSCORE_THRESHOLD:
                anomalies.append({
                    "anomaly": "session_length",
                    "severity": "high",
                    "detail": f"Session has {n_events} events "
                              f"(z-score: {z_score:.2f})",
                })

        # Check 2: Novel tool usage
        for event in current_session_events:
            tool = event.get("tool_name")
            if tool and tool not in profile.tool_call_frequency:
                anomalies.append({
                    "anomaly": "novel_tool",
                    "severity": "critical",
                    "detail": f"Agent invoked tool '{tool}' "
                              f"never seen in baseline",
                })

        # Check 3: Unknown data sink
        for event in current_session_events:
            tool = event.get("tool_name", "")
            if tool in ("send_email", "write_file", "http_post", "api_call"):
                target = event.get("metadata", {}).get("target", "")
                if target and target not in profile.known_data_sinks:
                    anomalies.append({
                        "anomaly": "unknown_data_sink",
                        "severity": "critical",
                        "detail": f"Data sent to unrecognized sink: "
                                  f"{target}",
                    })

        return anomalies

Exfiltration Pattern Detection¶

Data exfiltration from agents follows recognizable patterns: data is read from an internal source, then transmitted to an external sink within the same session or delegation chain. Detection requires correlating read and write operations:

python
class ExfiltrationDetector:
    """Detects potential data exfiltration by correlating data
    reads from internal sources with writes to external sinks."""

    INTERNAL_READ_TOOLS = {
        "database_query", "file_read", "vector_search",
        "secret_read", "internal_api_call",
    }
    EXTERNAL_WRITE_TOOLS = {
        "http_request", "send_email", "api_call_external",
        "webhook_post", "file_upload",
    }

    def __init__(self, time_window_seconds: int = 300):
        self.time_window = time_window_seconds

    def analyze_session(self, events: list[dict]) -> list[dict]:
        """Scan a session for read-then-exfiltrate patterns."""
        alerts = []
        reads = [
            e for e in events
            if e.get("tool_name") in self.INTERNAL_READ_TOOLS
        ]
        writes = [
            e for e in events
            if e.get("tool_name") in self.EXTERNAL_WRITE_TOOLS
        ]

        for write_event in writes:
            write_ts = write_event.get("timestamp_epoch", 0)
            preceding_reads = [
                r for r in reads
                if 0 < (write_ts - r.get("timestamp_epoch", 0))
                   <= self.time_window
            ]
            if preceding_reads:
                # Check if data volume exiting exceeds a threshold
                read_hashes = {
                    r.get("tool_result_hash") for r in preceding_reads
                }
                write_param_hash = write_event.get("tool_parameters_hash")

                alerts.append({
                    "alert_type": "potential_exfiltration",
                    "severity": "critical",
                    "session_id": write_event.get("session_id"),
                    "agent_id": write_event.get("agent_id"),
                    "write_tool": write_event.get("tool_name"),
                    "write_target": write_event.get(
                        "metadata", {}
                    ).get("target", "unknown"),
                    "preceding_read_count": len(preceding_reads),
                    "read_tools": [
                        r.get("tool_name") for r in preceding_reads
                    ],
                    "time_span_seconds": (
                        write_ts
                        - min(r.get("timestamp_epoch", 0)
                              for r in preceding_reads)
                    ),
                })
        return alerts

Meta-Detection: Detecting Detection Evasion¶

Sophisticated attackers will attempt to evade your detection systems. Meta-detection monitors for behaviors that suggest active evasion:

  • Log suppression — An agent that reduces its logging verbosity mid-session or strips metadata from events.
  • Timing manipulation — Artificially spacing tool calls to stay below rate-based anomaly thresholds.
  • Encoding obfuscation — Using Base64, Unicode substitutions, or character-level splitting to evade signature detection.
  • Tool aliasing — Calling a benign tool wrapper that internally delegates to a sensitive tool, bypassing tool-name-based alerting.
python
class MetaDetector:
    """Detects behaviors that suggest active evasion of
    security monitoring."""

    def detect_log_suppression(
        self, events: list[dict], expected_rate: float
    ) -> bool:
        """Alert when event emission rate drops significantly
        below the expected baseline."""
        if not events:
            return True  # No events at all is suspicious
        duration = (
            events[-1].get("timestamp_epoch", 0)
            - events[0].get("timestamp_epoch", 0)
        )
        if duration <= 0:
            return False
        actual_rate = len(events) / duration
        return actual_rate < (expected_rate * 0.3)

    def detect_field_stripping(self, events: list[dict]) -> list[dict]:
        """Detect events with suspiciously missing security
        fields that should always be present."""
        required_fields = [
            "agent_id", "session_id", "trace_id", "tool_name",
        ]
        suspicious = []
        for event in events:
            missing = [
                f for f in required_fields if not event.get(f)
            ]
            if missing:
                suspicious.append({
                    "event": event,
                    "missing_fields": missing,
                    "alert": "Possible field stripping evasion",
                })
        return suspicious

Key Takeaway Detection engineering for AI agents requires layered defenses: static signatures catch known prompt injection patterns, behavioral anomaly detection identifies novel attacks, exfiltration correlators track data flow across read-write pairs, and meta-detectors watch for active evasion of your monitoring itself. No single layer is sufficient—compose them into a detection pipeline where each layer compensates for the blind spots of the others.

15.3 Incident Response for Agent Compromises¶

When detection fires, you need a well-rehearsed incident response (IR) process tailored to the unique characteristics of agentic AI systems. Traditional IR playbooks assume human adversaries manipulating static software. Agent compromises introduce a fundamentally different dynamic: the compromised entity can reason, adapt, and potentially resist containment.

IR Playbook Structure for Agent Compromises¶

An effective agent IR playbook addresses six phases, adapted from NIST SP 800-61 but extended for agentic systems:

  1. Preparation — Tooling, permissions, and runbooks pre-staged for rapid response.
  2. Detection & Analysis — Triage alerts from the detection pipeline, determine scope of compromise.
  3. Containment — Isolate the affected agent(s) without disrupting uninvolved operations.
  4. Eradication — Remove the adversarial influence from the agent's context and data pipeline.
  5. Recovery — Restore agents to a known-good state with validated configurations.
  6. Post-Incident Activity — Analyze root cause, update detections, improve defenses.
yaml

Containment Strategies¶

Agent containment is more nuanced than terminating a process. The following strategies are ordered from least to most disruptive:

StrategyDescriptionWhen to Use
Tool RestrictionRevoke specific tool permissions while keeping the agent operationalLow-severity alerts; suspected but unconfirmed compromise
Context QuarantineFlush the agent's context window and reinitialize from checkpointConfirmed prompt injection; context is contaminated
Session TerminationEnd the agent's active session and all in-flight tasksActive exploitation observed; data exfiltration in progress
Cascade SuspensionSuspend the compromised agent and all agents in its delegation treeMulti-agent compromise; lateral movement suspected

Forensic Analysis of Agent Logs¶

Post-incident forensics for agent compromises requires reconstructing the causal chain from telemetry. The following script automates initial forensic triage:

python
import json
from datetime import datetime, timezone


class AgentForensicAnalyzer:
    """Automated forensic analysis for agent security incidents."""

    def __init__(self, events: list[dict]):
        self.events = sorted(
            events, key=lambda e: e.get("timestamp", "")
        )

    def reconstruct_timeline(self) -> list[dict]:
        """Build a chronological timeline of all agent actions
        with security annotations."""
        timeline = []
        for event in self.events:
            entry = {
                "timestamp": event.get("timestamp"),
                "agent_id": event.get("agent_id"),
                "action": event.get("action"),
                "tool_name": event.get("tool_name"),
                "severity": event.get("severity", "info"),
                "security_flags": [],
            }
            if event.get("policy_violations"):
                entry["security_flags"].append("POLICY_VIOLATION")
            if event.get("tool_name") in (
                "execute_shell", "http_request", "credential_read"
            ):
                entry["security_flags"].append("SENSITIVE_TOOL")
            if event.get("data_classification") in (
                "confidential", "restricted"
            ):
                entry["security_flags"].append("SENSITIVE_DATA")
            timeline.append(entry)
        return timeline

    def identify_injection_point(self) -> dict | None:
        """Attempt to identify the event where adversarial
        input first entered the agent's context."""
        for i, event in enumerate(self.events):
            if event.get("policy_violations"):
                preceding = self.events[max(0, i - 10): i]
                data_sources = [
                    e for e in preceding
                    if e.get("event_type") in (
                        "tool_invocation", "llm_interaction"
                    )
                    and e.get("tool_name") in (
                        "web_search", "document_read", "api_call",
                        "vector_search", "user_input",
                    )
                ]
                if data_sources:
                    return {
                        "suspected_injection_event": data_sources[-1],
                        "first_violation_event": event,
                        "events_between": i - self.events.index(
                            data_sources[-1]
                        ),
                    }
        return None

    def extract_iocs(self) -> dict:
        """Extract Indicators of Compromise from the incident
        event stream."""
        iocs = {
            "suspicious_targets": set(),
            "novel_tools_used": set(),
            "data_accessed": [],
            "external_endpoints": set(),
        }
        for event in self.events:
            tool = event.get("tool_name", "")
            target = event.get("metadata", {}).get("target", "")

            if tool in ("http_request", "api_call_external"):
                if target:
                    iocs["external_endpoints"].add(target)
            if event.get("severity") == "critical":
                if tool:
                    iocs["novel_tools_used"].add(tool)
            if event.get("data_classification") in (
                "confidential", "restricted"
            ):
                iocs["data_accessed"].append({
                    "tool": tool,
                    "timestamp": event.get("timestamp"),
                    "classification": event.get("data_classification"),
                })
            if target and tool in ("send_email", "webhook_post"):
                iocs["suspicious_targets"].add(target)

        iocs["suspicious_targets"] = list(iocs["suspicious_targets"])
        iocs["novel_tools_used"] = list(iocs["novel_tools_used"])
        iocs["external_endpoints"] = list(iocs["external_endpoints"])
        return iocs

    def generate_report(self) -> str:
        """Produce a structured forensic report."""
        timeline = self.reconstruct_timeline()
        injection = self.identify_injection_point()
        iocs = self.extract_iocs()

        report = {
            "report_type": "agent_forensic_analysis",
            "generated_at": datetime.now(timezone.utc).isoformat(),
            "total_events_analyzed": len(self.events),
            "timeline_summary": {
                "first_event": (
                    timeline[0]["timestamp"] if timeline else None
                ),
                "last_event": (
                    timeline[-1]["timestamp"] if timeline else None
                ),
                "flagged_events": sum(
                    1 for t in timeline if t["security_flags"]
                ),
            },
            "injection_analysis": injection,
            "indicators_of_compromise": iocs,
            "recommendations": self._generate_recommendations(
                injection, iocs
            ),
        }
        return json.dumps(report, indent=2, default=str)

    def _generate_recommendations(
        self, injection: dict | None, iocs: dict
    ) -> list[str]:
        recs = []
        if injection:
            source_tool = injection["suspected_injection_event"].get(
                "tool_name", ""
            )
            recs.append(
                f"Review input sanitization for tool: {source_tool}"
            )
            recs.append(
                "Add detection signature for this injection pattern"
            )
        if iocs["external_endpoints"]:
            recs.append(
                "Block the following external endpoints: "
                + ", ".join(iocs["external_endpoints"])
            )
        if iocs["data_accessed"]:
            recs.append(
                "Audit data access — notify data owners of potential "
                "exposure for classified data"
            )
        recs.append(
            "Rotate all credentials accessible to compromised agent"
        )
        return recs

Recovery Procedures¶

Recovery follows a strict sequence to prevent re-compromise:

  1. Validate the source configuration — Ensure the agent's manifest, system prompt, and tool allowlist have not been tampered with in the configuration repository.
  2. Rebuild, don't patch — Redeploy agents from immutable artifacts rather than attempting to "fix" a running instance. Containerized agents should be replaced from a clean image.
  3. Verify data pipeline integrity — If the agent reads from a RAG pipeline, verify that the document store and vector embeddings have not been poisoned.
  4. Graduated re-enablement — Bring the agent back online with enhanced monitoring. Reduce permissions initially and expand them over a 24–48 hour observation period if no anomalies are detected.
  5. Validate with adversarial probes — Run your prompt injection test suite against the restored agent to confirm defenses are effective.

Post-Incident Review¶

The post-incident review for agent compromises should answer questions that go beyond traditional incident retrospectives:

  • What was the injection vector? How did adversarial content enter the agent's context? Was it via user input, a RAG document, an API response, or inter-agent communication?
  • Why did the detection pipeline miss or delay alerting? Was it a gap in signature coverage, a behavioral baseline that was too permissive, or log data that was incomplete?
  • What was the full blast radius? Which agents were affected? What data was accessed or exfiltrated? Were downstream systems impacted?
  • Could the agent have contained itself? If the agent has self-monitoring capabilities, why did they not trigger? Should guardrails be hardened?
  • What organizational changes are needed? Do we need new policies for tool access? Should we change the agent's delegation authority?

Document findings in a structured format that feeds directly back into your detection engineering pipeline, creating a continuous improvement loop.

Key Takeaway Incident response for agent compromises demands purpose-built playbooks that account for cascading delegation chains, context contamination, and the agent's ability to reason about its own containment. Pre-stage containment actions, preserve forensic evidence before any remediation, and always rebuild from known-good artifacts rather than patching compromised instances in place.

Step 2: Real-Time Detection Engine¶

The detection engine consumes events from Kafka and applies the detection rules developed in Section 15.2:

python
import json
import asyncio
from dataclasses import dataclass
from aiokafka import AIOKafkaConsumer, AIOKafkaProducer


@dataclass
class Alert:
    alert_id: str
    alert_type: str
    severity: str
    agent_id: str
    session_id: str
    trace_id: str
    details: dict
    timestamp: str


class AgentSOCDetectionEngine:
    """Real-time detection engine that processes agent telemetry
    from Kafka and emits security alerts."""

    def __init__(
        self,
        kafka_brokers: list[str],
        input_topic: str = "agent-security-events",
        alert_topic: str = "agent-security-alerts",
    ):
        self.kafka_brokers = kafka_brokers
        self.input_topic = input_topic
        self.alert_topic = alert_topic

        # Initialize detection components
        self.injection_detector = PromptInjectionDetector()
        self.anomaly_detector = AgentAnomalyDetector()
        self.exfil_detector = ExfiltrationDetector()
        self.meta_detector = MetaDetector()

        # Session event buffers for stateful detection
        self.session_buffers: dict[str, list[dict]] = {}

    async def start(self):
        """Main event processing loop."""
        consumer = AIOKafkaConsumer(
            self.input_topic,
            bootstrap_servers=self.kafka_brokers,
            value_deserializer=lambda m: json.loads(m.decode()),
            group_id="agent-soc-detection",
            auto_offset_reset="latest",
        )
        self.producer = AIOKafkaProducer(
            bootstrap_servers=self.kafka_brokers,
            value_serializer=lambda v: json.dumps(v).encode(),
        )

        await consumer.start()
        await self.producer.start()

        try:
            async for message in consumer:
                event = message.value
                await self._process_event(event)
        finally:
            await consumer.stop()
            await self.producer.stop()

    async def _process_event(self, event: dict):
        """Apply all detection layers to an incoming event."""
        session_id = event.get("session_id", "unknown")

        # Buffer events by session for stateful detection
        if session_id not in self.session_buffers:
            self.session_buffers[session_id] = []
        self.session_buffers[session_id].append(event)

        # Layer 1: Prompt injection signatures
        if event.get("event_type") == "llm_interaction":
            prompt_text = event.get("metadata", {}).get(
                "prompt_text", ""
            )
            if prompt_text:
                detections = self.injection_detector.detect(prompt_text)
                for d in detections:
                    if d.detected:
                        await self._emit_alert(
                            alert_type="prompt_injection",
                            severity=d.threat_level.value,
                            event=event,
                            details={
                                "rule_id": d.rule_id,
                                "description": d.description,
                                "confidence": d.confidence,
                            },
                        )

        # Layer 2: Behavioral anomaly detection
        if event.get("event_type") == "tool_invocation":
            agent_id = event.get("agent_id", "")
            anomalies = self.anomaly_detector.check_anomalies(
                agent_id, self.session_buffers[session_id]
            )
            for anomaly in anomalies:
                if anomaly.get("severity") in ("high", "critical"):
                    await self._emit_alert(
                        alert_type=f"anomaly_{anomaly['anomaly']}",
                        severity=anomaly["severity"],
                        event=event,
                        details=anomaly,
                    )

        # Layer 3: Exfiltration pattern detection
        exfil_alerts = self.exfil_detector.analyze_session(
            self.session_buffers[session_id]
        )
        for exfil in exfil_alerts:
            await self._emit_alert(
                alert_type="potential_exfiltration",
                severity="critical",
                event=event,
                details=exfil,
            )

        # Layer 4: Meta-detection (evasion monitoring)
        stripped = self.meta_detector.detect_field_stripping(
            [event]
        )
        for s in stripped:
            await self._emit_alert(
                alert_type="evasion_field_stripping",
                severity="high",
                event=event,
                details=s,
            )

    async def _emit_alert(
        self,
        alert_type: str,
        severity: str,
        event: dict,
        details: dict,
    ):
        """Publish a security alert to the alerts topic."""
        import uuid
        from datetime import datetime, timezone

        alert = {
            "alert_id": str(uuid.uuid4()),
            "alert_type": alert_type,
            "severity": severity,
            "agent_id": event.get("agent_id"),
            "session_id": event.get("session_id"),
            "trace_id": event.get("trace_id"),
            "details": details,
            "source_event": event,
            "timestamp": datetime.now(timezone.utc).isoformat(),
        }
        await self.producer.send(self.alert_topic, alert)

Step 4: Automated Response Orchestrator¶

The response automator consumes alerts and executes pre-defined containment actions without human intervention for high-confidence, high-severity scenarios:

python
import json
import asyncio
import logging
from dataclasses import dataclass
from aiokafka import AIOKafkaConsumer

logger = logging.getLogger("agent_soc.response")


@dataclass
class ResponseAction:
    action: str
    target: str
    parameters: dict
    executed: bool = False
    result: str = ""


class AgentSOCResponseAutomator:
    """Automated incident response that executes containment
    actions based on alert severity and confidence."""

    # Thresholds for automated response without human approval
    AUTO_RESPONSE_THRESHOLDS = {
        "critical": {"min_confidence": 0.85},
        "high": {"min_confidence": 0.95},
    }

    def __init__(self, kafka_brokers: list[str], agent_control_api: str):
        self.kafka_brokers = kafka_brokers
        self.agent_control_api = agent_control_api
        self.action_log: list[ResponseAction] = []

    async def start(self):
        """Consume alerts and execute automated responses."""
        consumer = AIOKafkaConsumer(
            "agent-security-alerts",
            bootstrap_servers=self.kafka_brokers,
            value_deserializer=lambda m: json.loads(m.decode()),
            group_id="agent-soc-response",
        )
        await consumer.start()
        try:
            async for message in consumer:
                alert = message.value
                await self._handle_alert(alert)
        finally:
            await consumer.stop()

    async def _handle_alert(self, alert: dict):
        """Determine and execute the appropriate response."""
        severity = alert.get("severity", "low")
        confidence = alert.get("details", {}).get("confidence", 0.0)
        alert_type = alert.get("alert_type", "")
        agent_id = alert.get("agent_id", "")
        session_id = alert.get("session_id", "")

        threshold = self.AUTO_RESPONSE_THRESHOLDS.get(severity)
        if not threshold or confidence < threshold["min_confidence"]:
            logger.info(
                "Alert %s below auto-response threshold — "
                "routing to analyst queue",
                alert.get("alert_id"),
            )
            await self._route_to_analyst(alert)
            return

        # Execute graduated response based on alert type
        if alert_type == "potential_exfiltration":
            actions = await self._respond_exfiltration(
                agent_id, session_id, alert
            )
        elif alert_type == "prompt_injection":
            actions = await self._respond_injection(
                agent_id, session_id, alert
            )
        elif alert_type.startswith("anomaly_"):
            actions = await self._respond_anomaly(
                agent_id, session_id, alert
            )
        else:
            actions = await self._respond_generic(
                agent_id, session_id, alert
            )

        self.action_log.extend(actions)
        logger.warning(
            "Automated response executed for alert %s: %d actions",
            alert.get("alert_id"),
            len(actions),
        )

    async def _respond_exfiltration(
        self, agent_id: str, session_id: str, alert: dict
    ) -> list[ResponseAction]:
        """Maximum response — immediate isolation."""
        actions = []

        # 1. Terminate session immediately
        action = await self._execute_action(
            "terminate_session",
            session_id,
            {"reason": "exfiltration_detected"},
        )
        actions.append(action)

        # 2. Revoke all credentials
        action = await self._execute_action(
            "revoke_credentials",
            agent_id,
            {"scope": "all", "reason": "exfiltration_detected"},
        )
        actions.append(action)

        # 3. Suspend all agents in the delegation chain
        trace_id = alert.get("trace_id", "")
        action = await self._execute_action(
            "cascade_suspend",
            trace_id,
            {"reason": "exfiltration_detected"},
        )
        actions.append(action)

        # 4. Preserve forensic evidence
        action = await self._execute_action(
            "forensic_snapshot",
            session_id,
            {"include_context_window": True},
        )
        actions.append(action)

        return actions

    async def _respond_injection(
        self, agent_id: str, session_id: str, alert: dict
    ) -> list[ResponseAction]:
        """Context quarantine and enhanced monitoring."""
        actions = []

        action = await self._execute_action(
            "quarantine_context",
            session_id,
            {"reinitialize_from": "last_clean_checkpoint"},
        )
        actions.append(action)

        action = await self._execute_action(
            "elevate_monitoring",
            agent_id,
            {"level": "maximum", "duration_hours": 24},
        )
        actions.append(action)

        return actions

    async def _respond_anomaly(
        self, agent_id: str, session_id: str, alert: dict
    ) -> list[ResponseAction]:
        """Restrict tools and increase monitoring."""
        actions = []

        action = await self._execute_action(
            "restrict_tools",
            agent_id,
            {
                "deny_list": [
                    "execute_shell", "http_request",
                    "api_call_external",
                ],
                "duration_hours": 4,
            },
        )
        actions.append(action)

        action = await self._execute_action(
            "elevate_monitoring",
            agent_id,
            {"level": "enhanced", "duration_hours": 48},
        )
        actions.append(action)

        return actions

    async def _respond_generic(
        self, agent_id: str, session_id: str, alert: dict
    ) -> list[ResponseAction]:
        """Default response — log and monitor."""
        action = await self._execute_action(
            "elevate_monitoring",
            agent_id,
            {"level": "enhanced", "duration_hours": 12},
        )
        return [action]

    async def _execute_action(
        self, action: str, target: str, parameters: dict
    ) -> ResponseAction:
        """Execute a containment action via the agent control API."""
        response_action = ResponseAction(
            action=action, target=target, parameters=parameters
        )
        try:
            # In production, this calls the agent control API
            logger.info(
                "Executing: %s on %s with %s",
                action, target, json.dumps(parameters),
            )
            response_action.executed = True
            response_action.result = "success"
        except Exception as e:
            response_action.result = f"failed: {e}"
            logger.error("Action failed: %s — %s", action, e)
        return response_action

    async def _route_to_analyst(self, alert: dict):
        """Send low-confidence alerts to the human analyst queue."""
        logger.info(
            "Routing alert %s (severity=%s) to analyst queue",
            alert.get("alert_id"),
            alert.get("severity"),
        )

Putting It All Together¶

To deploy the Agent SOC:

  1. Deploy the telemetry pipeline — Start the OpenTelemetry Collector with the configuration above. Ensure all agents in your fleet are instrumented with the AgentSecurityLogger from Section 15.1.
  2. Initialize behavioral baselines — Run the AgentAnomalyDetector.build_profile() against at least 30 days of historical telemetry for each agent in your fleet.
  3. Start the detection engine — Deploy AgentSOCDetectionEngine as a Kafka consumer group. Scale horizontally by adding consumer instances; Kafka partitioning ensures each event is processed exactly once.
  4. Configure the dashboard — Import the Grafana dashboard JSON into your Grafana instance. Connect it to your Elasticsearch data source.
  5. Enable response automation — Deploy AgentSOCResponseAutomator with conservative confidence thresholds initially. Monitor the action log for false positives and adjust thresholds over a two-week burn-in period.
  6. Conduct a tabletop exercise — Before going live, run a simulated agent compromise through the entire pipeline. Verify that telemetry flows correctly, detections fire, the dashboard updates, and automated responses execute as expected.

Key Takeaway An Agent Security Operations Center is not a single tool—it is an integrated pipeline spanning telemetry ingestion, real-time detection, visual monitoring, and automated response. Build it incrementally: start with structured logging and manual analysis, then layer in automated detection, dashboards, and response automation as your operational maturity grows. The SOC is only as good as the telemetry it consumes—invest in instrumentation before investing in dashboards.
