Chapter 18: Building CLI Tools and APIs¶ ~16 min read

CLI tools and APIs are Claude Code’s sweet spot. No UI rendering, no browser quirks, no CSS — just structured input, logic, and structured output. This is the domain where Claude Code produces the most reliable, production-ready code with the least correction.

This chapter walks through building two real projects: a Python CLI tool for git repository analytics, and a FastAPI notification service. We finish with a brief look at GraphQL APIs. Every code example is complete, not a stub.

Part 1: Building a Python CLI Tool¶

Building the CLI Entry Point¶

Create the CLI entry point with a Click command group.
Include global options: --repo (path to git repo, defaults to current directory),
--json (output as JSON instead of rich tables), --verbose.
python
# src/git_analytics/cli.py

from __future__ import annotations

import sys
from pathlib import Path

import click

from git_analytics.core.git import GitRepo, GitError


class Context:
    """Shared context passed to all commands."""

    def __init__(self, repo_path: Path, json_output: bool, verbose: bool) -> None:
        self.repo_path = repo_path
        self.json_output = json_output
        self.verbose = verbose
        self._repo: GitRepo | None = None

    @property
    def repo(self) -> GitRepo:
        if self._repo is None:
            self._repo = GitRepo(self.repo_path)
        return self._repo


pass_context = click.make_pass_decorator(Context)


@click.group()
@click.option(
    "--repo",
    type=click.Path(exists=True, file_okay=False, path_type=Path),
    default=Path("."),
    help="Path to git repository.",
)
@click.option("--json", "json_output", is_flag=True, help="Output as JSON.")
@click.option("--verbose", "-v", is_flag=True, help="Verbose output.")
@click.version_option()
@click.pass_context
def cli(ctx: click.Context, repo: Path, json_output: bool, verbose: bool) -> None:
    """Analyze git repository statistics."""
    ctx.ensure_object(dict)
    try:
        ctx.obj = Context(repo_path=repo, json_output=json_output, verbose=verbose)
    except GitError as e:
        click.echo(f"Error: {e}", err=True)
        sys.exit(1)

Building the Core Git Module¶

The core module handles all git interactions. No Click dependency here — this is pure logic that can be tested independently.

Build the core git module that reads repository data.
Use subprocess to call git (not a Python git library).
Parse git log output into structured data.
python
# src/git_analytics/core/git.py

from __future__ import annotations

import subprocess
from collections import Counter, defaultdict
from dataclasses import dataclass
from datetime import datetime
from pathlib import Path


class GitError(Exception):
    """Raised when git operations fail."""


@dataclass
class Commit:
    hash: str
    author: str
    email: str
    date: datetime
    message: str
    files_changed: int
    insertions: int
    deletions: int


@dataclass
class FileChange:
    path: str
    insertions: int
    deletions: int


class GitRepo:
    """Interface to a git repository via subprocess calls."""

    def __init__(self, path: Path) -> None:
        self.path = path.resolve()
        if not (self.path / ".git").is_dir():
            raise GitError(f"Not a git repository: {self.path}")

    def _run(self, *args: str) -> str:
        """Execute a git command and return stdout."""
        try:
            result = subprocess.run(
                ["git", *args],
                cwd=self.path,
                capture_output=True,
                text=True,
                check=True,
                timeout=60,
            )
            return result.stdout.strip()
        except subprocess.CalledProcessError as e:
            raise GitError(f"git {' '.join(args)} failed: {e.stderr.strip()}") from e
        except subprocess.TimeoutExpired as e:
            raise GitError(f"git command timed out: {' '.join(args)}") from e

    def get_commits(self, since: str | None = None, limit: int = 1000) -> list[Commit]:
        """Get commit history with stats."""
        cmd = [
            "log",
            f"--max-count={limit}",
            "--format=%H|%an|%ae|%aI|%s",
            "--shortstat",
        ]
        if since:
            cmd.append(f"--since={since}")

        raw = self._run(*cmd)
        return self._parse_log(raw)

    def get_file_changes(
        self, since: str | None = None, limit: int = 1000
    ) -> dict[str, list[FileChange]]:
        """Get per-file change statistics grouped by commit."""
        cmd = [
            "log",
            f"--max-count={limit}",
            "--format=%H",
            "--numstat",
        ]
        if since:
            cmd.append(f"--since={since}")

        raw = self._run(*cmd)
        return self._parse_numstat(raw)

    def get_hotspots(self, limit: int = 20, since: str | None = None) -> list[tuple[str, int]]:
        """Find most frequently changed files."""
        changes = self.get_file_changes(since=since)
        counter: Counter[str] = Counter()
        for file_changes in changes.values():
            for fc in file_changes:
                if not fc.path.startswith("vendor/") and fc.path != "":
                    counter[fc.path] += 1
        return counter.most_common(limit)

    def get_churn(self, limit: int = 20, since: str | None = None) -> list[tuple[str, int, int]]:
        """Find files with most code churn (insertions + deletions)."""
        changes = self.get_file_changes(since=since)
        churn: defaultdict[str, tuple[int, int]] = defaultdict(lambda: (0, 0))

        for file_changes in changes.values():
            for fc in file_changes:
                if fc.path:
                    ins, dels = churn[fc.path]
                    churn[fc.path] = (ins + fc.insertions, dels + fc.deletions)

        sorted_churn = sorted(
            churn.items(),
            key=lambda x: x[1][0] + x[1][1],
            reverse=True,
        )
        return [(path, ins, dels) for path, (ins, dels) in sorted_churn[:limit]]

    def get_contributors(self, since: str | None = None) -> list[dict[str, int | str]]:
        """Get contributor statistics."""
        commits = self.get_commits(since=since)
        stats: defaultdict[str, dict[str, int]] = defaultdict(
            lambda: {"commits": 0, "insertions": 0, "deletions": 0}
        )

        for commit in commits:
            s = stats[commit.author]
            s["commits"] += 1
            s["insertions"] += commit.insertions
            s["deletions"] += commit.deletions

        result = []
        for author, s in sorted(stats.items(), key=lambda x: x[1]["commits"], reverse=True):
            result.append({"author": author, **s})
        return result

    def _parse_log(self, raw: str) -> list[Commit]:
        """Parse git log --format=%H|%an|%ae|%aI|%s --shortstat output."""
        commits: list[Commit] = []
        lines = raw.split("\n")
        i = 0

        while i < len(lines):
            line = lines[i].strip()
            if not line or "|" not in line:
                i += 1
                continue

            parts = line.split("|", maxsplit=4)
            if len(parts) < 5:
                i += 1
                continue

            hash_, author, email, date_str, message = parts
            files_changed = insertions = deletions = 0

            # Look for shortstat line
            if i + 1 < len(lines):
                stat_line = lines[i + 1].strip()
                if "file" in stat_line:
                    files_changed, insertions, deletions = self._parse_shortstat(stat_line)
                    i += 1

            commits.append(
                Commit(
                    hash=hash_,
                    author=author,
                    email=email,
                    date=datetime.fromisoformat(date_str),
                    message=message,
                    files_changed=files_changed,
                    insertions=insertions,
                    deletions=deletions,
                )
            )
            i += 1

        return commits

    @staticmethod
    def _parse_shortstat(line: str) -> tuple[int, int, int]:
        """Parse a git shortstat line like '3 files changed, 10 insertions(+), 2 deletions(-)'."""
        files = insertions = deletions = 0
        for part in line.split(","):
            part = part.strip()
            if "file" in part:
                files = int(part.split()[0])
            elif "insertion" in part:
                insertions = int(part.split()[0])
            elif "deletion" in part:
                deletions = int(part.split()[0])
        return files, insertions, deletions

    def _parse_numstat(self, raw: str) -> dict[str, list[FileChange]]:
        """Parse git log --numstat output into per-commit file changes."""
        result: dict[str, list[FileChange]] = {}
        current_hash = ""
        lines = raw.split("\n")

        for line in lines:
            line = line.strip()
            if not line:
                continue
            if len(line) == 40 and all(c in "0123456789abcdef" for c in line):
                current_hash = line
                result[current_hash] = []
            elif "\t" in line and current_hash:
                parts = line.split("\t")
                if len(parts) == 3:
                    ins_str, del_str, path = parts
                    ins = int(ins_str) if ins_str != "-" else 0
                    dels = int(del_str) if del_str != "-" else 0
                    result[current_hash].append(FileChange(path=path, insertions=ins, deletions=dels))

        return result

Building Commands¶

Now we build each command. Here is the prompt and output for the stats command:

Add a 'stats' command that shows commit frequency grouped by day, week, or month.
Include a --since option (e.g., "3 months ago") and a --group-by option (day/week/month).
Output as a Rich table or JSON based on the global --json flag.
python
# src/git_analytics/commands/stats.py

from __future__ import annotations

import json
from collections import Counter
from datetime import datetime

import click
from rich.console import Console
from rich.table import Table

from git_analytics.cli import cli, pass_context, Context


@cli.command()
@click.option(
    "--since",
    default="6 months ago",
    help="Show stats since this date (e.g., '3 months ago', '2024-01-01').",
)
@click.option(
    "--group-by",
    type=click.Choice(["day", "week", "month"]),
    default="week",
    help="Group commits by time period.",
)
@pass_context
def stats(ctx: Context, since: str, group_by: str) -> None:
    """Show commit frequency over time."""
    commits = ctx.repo.get_commits(since=since)

    if not commits:
        click.echo("No commits found in the specified range.", err=True)
        return

    groups = _group_commits(commits, group_by)
    total_commits = len(commits)
    total_insertions = sum(c.insertions for c in commits)
    total_deletions = sum(c.deletions for c in commits)

    if ctx.json_output:
        data = {
            "summary": {
                "total_commits": total_commits,
                "total_insertions": total_insertions,
                "total_deletions": total_deletions,
                "period": since,
                "group_by": group_by,
            },
            "groups": [
                {"period": period, "commits": count}
                for period, count in sorted(groups.items())
            ],
        }
        click.echo(json.dumps(data, indent=2))
        return

    console = Console()
    console.print(f"\n[bold]Commit Statistics[/bold] (since {since})\n")
    console.print(
        f"Total: [green]{total_commits}[/green] commits, "
        f"[green]+{total_insertions}[/green] / [red]-{total_deletions}[/red] lines\n"
    )

    table = Table(show_header=True, header_style="bold cyan")
    table.add_column("Period", style="dim")
    table.add_column("Commits", justify="right")
    table.add_column("Graph", min_width=30)

    max_count = max(groups.values()) if groups else 1
    for period, count in sorted(groups.items()):
        bar_length = int((count / max_count) * 30)
        bar = "█" * bar_length
        table.add_row(period, str(count), f"[green]{bar}[/green]")

    console.print(table)


def _group_commits(commits: list, group_by: str) -> Counter[str]:
    """Group commits by time period and count them."""
    counter: Counter[str] = Counter()
    for commit in commits:
        dt: datetime = commit.date
        if group_by == "day":
            key = dt.strftime("%Y-%m-%d")
        elif group_by == "week":
            key = f"{dt.isocalendar()[0]}-W{dt.isocalendar()[1]:02d}"
        elif group_by == "month":
            key = dt.strftime("%Y-%m")
        else:
            key = dt.strftime("%Y-%m-%d")
        counter[key] += 1
    return counter

The hotspots command:

Add a 'hotspots' command that finds the most frequently changed files.
Include --limit (default 20) and --since options.
python
# src/git_analytics/commands/hotspots.py

from __future__ import annotations

import json

import click
from rich.console import Console
from rich.table import Table

from git_analytics.cli import cli, pass_context, Context


@cli.command()
@click.option("--limit", "-n", default=20, help="Number of files to show.")
@click.option("--since", default=None, help="Only consider commits since this date.")
@pass_context
def hotspots(ctx: Context, limit: int, since: str | None) -> None:
    """Find the most frequently changed files (bug magnets)."""
    results = ctx.repo.get_hotspots(limit=limit, since=since)

    if not results:
        click.echo("No file changes found.", err=True)
        return

    if ctx.json_output:
        data = [{"file": path, "changes": count} for path, count in results]
        click.echo(json.dumps(data, indent=2))
        return

    console = Console()
    console.print("\n[bold]File Hotspots[/bold] (most frequently changed files)\n")

    table = Table(show_header=True, header_style="bold cyan")
    table.add_column("#", justify="right", style="dim", width=4)
    table.add_column("File")
    table.add_column("Changes", justify="right")
    table.add_column("Frequency", min_width=20)

    max_count = results[0][1] if results else 1
    for i, (path, count) in enumerate(results, 1):
        bar_length = int((count / max_count) * 20)
        bar = "█" * bar_length
        heat = "red" if i <= 3 else "yellow" if i <= 10 else "green"
        table.add_row(str(i), path, str(count), f"[{heat}]{bar}[/{heat}]")

    console.print(table)

from __future__ import annotations

from datetime import datetime
from enum import StrEnum

from pydantic import BaseModel, Field


class NotificationType(StrEnum):
    EMAIL = "email"
    SMS = "sms"
    PUSH = "push"
    WEBHOOK = "webhook"


class NotificationStatus(StrEnum):
    PENDING = "pending"
    SENT = "sent"
    FAILED = "failed"
    DELIVERED = "delivered"


class NotificationCreate(BaseModel):
    """Request body for creating a notification."""

    recipient: str = Field(..., min_length=1, max_length=500)
    type: NotificationType
    subject: str = Field(..., min_length=1, max_length=200)
    body: str = Field(..., min_length=1, max_length=10000)
    metadata: dict[str, str] = Field(default_factory=dict)


class Notification(BaseModel):
    """Full notification model returned from the API."""

    id: str
    recipient: str
    type: NotificationType
    subject: str
    body: str
    status: NotificationStatus
    metadata: dict[str, str]
    created_at: datetime
    sent_at: datetime | None = None
    error: str | None = None


class NotificationList(BaseModel):
    """Paginated list of notifications."""

    data: list[Notification]
    total: int
    page: int
    limit: int
    has_more: bool

# src/notifications/database.py

from __future__ import annotations

import json
import uuid
from datetime import datetime, timezone
from pathlib import Path

import aiosqlite

from notifications.models import (
    Notification,
    NotificationCreate,
    NotificationStatus,
)

DATABASE_PATH = Path("data/notifications.db")


async def get_db() -> aiosqlite.Connection:
    """Get a database connection."""
    DATABASE_PATH.parent.mkdir(parents=True, exist_ok=True)
    db = await aiosqlite.connect(DATABASE_PATH)
    db.row_factory = aiosqlite.Row
    await db.execute("PRAGMA journal_mode=WAL")
    await db.execute("PRAGMA foreign_keys=ON")
    return db


async def init_db() -> None:
    """Initialize the database schema."""
    db = await get_db()
    try:
        await db.execute("""
            CREATE TABLE IF NOT EXISTS notifications (
                id TEXT PRIMARY KEY,
                recipient TEXT NOT NULL,
                type TEXT NOT NULL,
                subject TEXT NOT NULL,
                body TEXT NOT NULL,
                status TEXT NOT NULL DEFAULT 'pending',
                metadata TEXT NOT NULL DEFAULT '{}',
                created_at TEXT NOT NULL,
                sent_at TEXT,
                error TEXT
            )
        """)
        await db.execute("""
            CREATE INDEX IF NOT EXISTS idx_notifications_status
            ON notifications(status)
        """)
        await db.execute("""
            CREATE INDEX IF NOT EXISTS idx_notifications_created
            ON notifications(created_at DESC)
        """)
        await db.commit()
    finally:
        await db.close()


async def create_notification(data: NotificationCreate) -> Notification:
    """Insert a new notification and return it."""
    db = await get_db()
    try:
        notification_id = str(uuid.uuid4())
        now = datetime.now(timezone.utc).isoformat()

        await db.execute(
            """
            INSERT INTO notifications (id, recipient, type, subject, body, status, metadata, created_at)
            VALUES (?, ?, ?, ?, ?, ?, ?, ?)
            """,
            (
                notification_id,
                data.recipient,
                data.type.value,
                data.subject,
                data.body,
                NotificationStatus.PENDING.value,
                json.dumps(data.metadata),
                now,
            ),
        )
        await db.commit()

        return Notification(
            id=notification_id,
            recipient=data.recipient,
            type=data.type,
            subject=data.subject,
            body=data.body,
            status=NotificationStatus.PENDING,
            metadata=data.metadata,
            created_at=datetime.fromisoformat(now),
        )
    finally:
        await db.close()


async def get_notification(notification_id: str) -> Notification | None:
    """Get a notification by ID."""
    db = await get_db()
    try:
        cursor = await db.execute(
            "SELECT * FROM notifications WHERE id = ?",
            (notification_id,),
        )
        row = await cursor.fetchone()
        if row is None:
            return None
        return _row_to_notification(row)
    finally:
        await db.close()


async def list_notifications(
    page: int = 1,
    limit: int = 20,
    status: NotificationStatus | None = None,
) -> tuple[list[Notification], int]:
    """List notifications with pagination and optional status filter."""
    db = await get_db()
    try:
        where_clause = ""
        params: list[str | int] = []

        if status:
            where_clause = "WHERE status = ?"
            params.append(status.value)

        # Get total count
        cursor = await db.execute(
            f"SELECT COUNT(*) FROM notifications {where_clause}",
            params,
        )
        row = await cursor.fetchone()
        total = row[0] if row else 0

        # Get page
        offset = (page - 1) * limit
        cursor = await db.execute(
            f"""
            SELECT * FROM notifications {where_clause}
            ORDER BY created_at DESC
            LIMIT ? OFFSET ?
            """,
            [*params, limit, offset],
        )
        rows = await cursor.fetchall()

        notifications = [_row_to_notification(r) for r in rows]
        return notifications, total
    finally:
        await db.close()


def _row_to_notification(row: aiosqlite.Row) -> Notification:
    """Convert a database row to a Notification model."""
    return Notification(
        id=row["id"],
        recipient=row["recipient"],
        type=row["type"],
        subject=row["subject"],
        body=row["body"],
        status=row["status"],
        metadata=json.loads(row["metadata"]),
        created_at=datetime.fromisoformat(row["created_at"]),
        sent_at=datetime.fromisoformat(row["sent_at"]) if row["sent_at"] else None,
        error=row["error"],
    )

@asynccontextmanager

    yield


app = FastAPI(
    title="Notification Service",
    version="1.0.0",
    lifespan=lifespan,
)

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_methods=["*"],
    allow_headers=["*"],
)


@app.post("/notifications", response_model=Notification, status_code=201)
async def send_notification(data: NotificationCreate) -> Notification:
    """Create and queue a new notification."""
    notification = await create_notification(data)

    # Broadcast to WebSocket subscribers
    await manager.broadcast(notification.model_dump(mode="json"))

    return notification


@app.get("/notifications/{notification_id}", response_model=Notification)
async def get_notification_by_id(notification_id: str) -> Notification:
    """Get a notification by its ID."""
    notification = await get_notification(notification_id)
    if notification is None:
        raise HTTPException(status_code=404, detail="Notification not found")
    return notification


@app.get("/notifications", response_model=NotificationList)
async def list_all_notifications(
    page: int = Query(default=1, ge=1),
    limit: int = Query(default=20, ge=1, le=100),
    status: NotificationStatus | None = Query(default=None),
) -> NotificationList:
    """List notifications with pagination."""
    notifications, total = await list_notifications(
        page=page, limit=limit, status=status
    )
    return NotificationList(
        data=notifications,
        total=total,
        page=page,
        limit=limit,
        has_more=(page * limit) < total,
    )


@app.websocket("/ws/subscribe")
async def websocket_subscribe(websocket: WebSocket) -> None:
    """Subscribe to real-time notification events."""
    await manager.connect(websocket)
    try:
        while True:
            # Keep connection alive, handle client messages if needed
            data = await websocket.receive_text()
            # Echo back as acknowledgment
            await websocket.send_json({"type": "ack", "data": data})
    except WebSocketDisconnect:
        manager.disconnect(websocket)

# src/notifications/ws.py

from __future__ import annotations

import json
from typing import Any

from fastapi import WebSocket


class ConnectionManager:
    """Manages WebSocket connections for real-time notification delivery."""

    def __init__(self) -> None:
        self.active_connections: list[WebSocket] = []

    async def connect(self, websocket: WebSocket) -> None:
        await websocket.accept()
        self.active_connections.append(websocket)

    def disconnect(self, websocket: WebSocket) -> None:
        self.active_connections.remove(websocket)

    async def broadcast(self, message: dict[str, Any]) -> None:
        """Send a message to all connected clients."""
        payload = json.dumps({"type": "notification", "data": message})
        disconnected: list[WebSocket] = []
        for connection in self.active_connections:
            try:
                await connection.send_text(payload)
            except Exception:
                disconnected.append(connection)

        for conn in disconnected:
            self.active_connections.remove(conn)


manager = ConnectionManager()

Part 3: Building a GraphQL API¶

GraphQL APIs follow a different development pattern with Claude Code. Instead of defining routes, you define a schema first and let Claude implement the resolvers.

Schema-First Approach¶

Here is my GraphQL schema. Implement the resolvers using Strawberry (Python GraphQL library) 
with the same SQLite database we used for the notification service.
graphql
type Query {
  notification(id: ID!): Notification
  notifications(page: Int = 1, limit: Int = 20, status: NotificationStatus): NotificationConnection!
}

type Mutation {
  sendNotification(input: NotificationInput!): Notification!
}

type Subscription {
  notificationCreated: Notification!
}

type Notification {
  id: ID!
  recipient: String!
  type: NotificationType!
  subject: String!
  body: String!
  status: NotificationStatus!
  createdAt: DateTime!
  sentAt: DateTime
}

input NotificationInput {
  recipient: String!
  type: NotificationType!
  subject: String!
  body: String!
}

type NotificationConnection {
  nodes: [Notification!]!
  totalCount: Int!
  hasMore: Boolean!
}

enum NotificationType { EMAIL SMS PUSH WEBHOOK }
enum NotificationStatus { PENDING SENT FAILED DELIVERED }

Claude excels at schema-to-code generation because the schema is an unambiguous contract. The resolver implementations follow directly from the schema types and the existing database layer.

Patterns That Work Well for CLI/API Development¶

Test-First with Claude¶

The most effective pattern for CLI and API development:

  1. Write the test (or tell Claude what the test should verify)
  2. Tell Claude to make the test pass
  3. Claude implements the minimum code to pass
Write a test that verifies the 'churn' command correctly identifies
files with the most insertions + deletions. Then make it pass.

Claude writes the test, runs it (it fails), implements the command, runs it again (it passes). This is faster than describing the implementation because tests are unambiguous specifications.

Schema-Driven Development¶

For APIs, define the interface first:

  • REST: write the OpenAPI spec or Pydantic models first
  • GraphQL: write the schema first
  • CLI: write the --help text first

Then tell Claude to implement against that specification. The interface acts as a contract that constrains Claude’s implementation choices.

Progressive Enhancement¶

Build in layers:

  1. Basic CRUD (happy path only)
  2. Validation and error handling
  3. Pagination and filtering
  4. Real-time features
  5. Performance optimization

Each layer is independently testable. You verify before moving to the next.

Use /plan for API Design Decisions¶

Before implementing, use /plan to think through the design:

/plan
Design the API for a notification service. Consider:
- Should we use REST or GraphQL?
- How should we handle retry logic for failed notifications?
- What's the pagination strategy?
- How do we handle webhook delivery?

Claude produces a design document. You review, adjust, then implement. This prevents mid-implementation pivots that waste time.

What Most People Get Wrong¶

Building everything in one prompt. “Build me a complete REST API for notifications” produces a monolithic result that is hard to review and hard to test. Work incrementally: models first, then one endpoint at a time, then tests for that endpoint.

Ignoring the --json flag for CLIs. Every CLI command should support machine-readable output. Claude includes this by default if you put it in your CLI rules, but most people forget to ask for it. Machine-readable output makes your CLI composable with other tools.

Mocking the database in API tests. Integration tests that hit a real database (even SQLite in-memory) catch real bugs. Mocked tests only verify that your mocks are correct. As discussed in Chapter 19, reserve mocks for third-party services you do not control.

Chapter Summary¶

  1. CLI tools and APIs are Claude Code’s strongest domain. Text in, text out. No rendering ambiguity. Claude produces production-ready code with minimal correction.

  2. Separate logic from interface. Core modules (git analysis, database operations) have no CLI or HTTP dependencies. This makes them independently testable and reusable.

  3. Test-first is the fastest workflow for CLIs and APIs. Write the test, tell Claude to make it pass. Tests are unambiguous specifications that eliminate back-and-forth.

  4. Schema-driven development constrains Claude’s choices. Define the interface (Pydantic models, GraphQL schema, CLI --help) first. Claude implements against it accurately.

  5. Build incrementally. One command at a time, one endpoint at a time, tests at each step. Verify before moving forward. This is faster than building everything at once and debugging the result.
