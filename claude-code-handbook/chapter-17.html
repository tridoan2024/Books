Chapter 17: Building a Full-Stack Web Application¶ ~18 min read

This chapter is a complete walkthrough. We are building a task management application — boards, lists, and cards, similar to Trello — from an empty directory to a deployed product. Every line of code is generated by Claude Code. Every prompt shown here is real. The goal is not to teach you React or Express; it is to show you how a senior engineer uses Claude Code to build a production application efficiently.

By the end of this chapter, you will have seen every phase of full-stack development through Claude Code: project scaffolding, database design, API development, frontend components, real-time features, testing, and deployment. More importantly, you will understand when to use /plan mode, when to prompt directly, and when to step back and let Claude Code run.

The Application: TaskFlow¶

TaskFlow is a task management SaaS with these features:

  • User authentication (JWT)
  • Boards with customizable lists
  • Draggable cards with descriptions, labels, and assignees
  • Real-time updates via WebSocket (other users see changes instantly)
  • Team collaboration with role-based access

The tech stack:

Layer Technology
Frontend React 18, TypeScript, Vite, Tailwind CSS
Backend Express.js, TypeScript
Database PostgreSQL, Prisma ORM
Auth JWT with refresh tokens
Real-time WebSocket (ws library)
Testing Vitest (frontend), Vitest (backend), Supertest
Deployment Docker, GitHub Actions

Phase 2: Database and API¶

Designing the Schema¶

Design the PostgreSQL schema for TaskFlow. We need:
- Users (email, password hash, name, avatar)
- Boards (title, owner, members with roles)
- Lists within boards (title, position for ordering)
- Cards within lists (title, description, position, assignee, labels, due date)
- Activity log for audit trail

Claude generates the complete Prisma schema:

prisma
// packages/api/src/prisma/schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id           String   @id @default(cuid())
  email        String   @unique
  passwordHash String   @map("password_hash")
  name         String
  avatarUrl    String?  @map("avatar_url")
  createdAt    DateTime @default(now()) @map("created_at")
  updatedAt    DateTime @updatedAt @map("updated_at")

  ownedBoards   Board[]        @relation("BoardOwner")
  memberships   BoardMember[]
  assignedCards Card[]         @relation("CardAssignee")
  activities    Activity[]

  @@map("users")
}

model Board {
  id        String   @id @default(cuid())
  title     String
  ownerId   String   @map("owner_id")
  createdAt DateTime @default(now()) @map("created_at")
  updatedAt DateTime @updatedAt @map("updated_at")

  owner      User          @relation("BoardOwner", fields: [ownerId], references: [id])
  members    BoardMember[]
  lists      List[]
  activities Activity[]

  @@map("boards")
}

enum BoardRole {
  ADMIN
  MEMBER
  VIEWER
}

model BoardMember {
  id      String    @id @default(cuid())
  boardId String    @map("board_id")
  userId  String    @map("user_id")
  role    BoardRole @default(MEMBER)

  board Board @relation(fields: [boardId], references: [id], onDelete: Cascade)
  user  User  @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([boardId, userId])
  @@map("board_members")
}

model List {
  id       String @id @default(cuid())
  title    String
  position Int
  boardId  String @map("board_id")

  board Board  @relation(fields: [boardId], references: [id], onDelete: Cascade)
  cards Card[]

  @@map("lists")
}

model Card {
  id          String    @id @default(cuid())
  title       String
  description String?
  position    Int
  dueDate     DateTime? @map("due_date")
  labels      String[]  @default([])
  listId      String    @map("list_id")
  assigneeId  String?   @map("assignee_id")
  createdAt   DateTime  @default(now()) @map("created_at")
  updatedAt   DateTime  @updatedAt @map("updated_at")

  list     List @relation(fields: [listId], references: [id], onDelete: Cascade)
  assignee User? @relation("CardAssignee", fields: [assigneeId], references: [id])

  @@map("cards")
}

model Activity {
  id        String   @id @default(cuid())
  action    String
  details   Json?
  userId    String   @map("user_id")
  boardId   String   @map("board_id")
  createdAt DateTime @default(now()) @map("created_at")

  user  User  @relation(fields: [userId], references: [id])
  board Board @relation(fields: [boardId], references: [id], onDelete: Cascade)

  @@index([boardId, createdAt])
  @@map("activities")
}

Notice Claude’s decisions: cuid() for IDs (URL-safe, sortable), @@map for snake_case table names (PostgreSQL convention while keeping camelCase in TypeScript), position fields for ordering, cascade deletes for owned entities, and a JSON field for flexible activity details.

Building the API¶

Now we build the API endpoints. Rather than one massive prompt, we work incrementally:

Build the board CRUD endpoints:
- POST /api/boards — create a board (creator becomes ADMIN)
- GET /api/boards — list boards the user is a member of
- GET /api/boards/:id — get board with lists and cards
- PATCH /api/boards/:id — update board title
- DELETE /api/boards/:id — delete board (ADMIN only)
All endpoints require authentication.
Include Zod validation for request bodies.

Claude generates the complete route handler, service, and validation:

typescript
// packages/api/src/routes/boards.ts

import { Router } from 'express';
import { z } from 'zod';
import { authenticate } from '../middleware/auth.js';
import { validate } from '../middleware/validate.js';
import * as boardService from '../services/board.service.js';

const router = Router();

const createBoardSchema = z.object({
  body: z.object({
    title: z.string().min(1).max(100),
  }),
});

const updateBoardSchema = z.object({
  body: z.object({
    title: z.string().min(1).max(100).optional(),
  }),
  params: z.object({
    id: z.string().cuid(),
  }),
});

router.use(authenticate);

router.post('/', validate(createBoardSchema), async (req, res) => {
  const board = await boardService.createBoard({
    title: req.body.title,
    ownerId: req.user!.id,
  });
  res.status(201).json({ data: board });
});

router.get('/', async (req, res) => {
  const boards = await boardService.getBoardsForUser(req.user!.id);
  res.json({ data: boards });
});

router.get('/:id', async (req, res) => {
  const board = await boardService.getBoardWithContents(
    req.params.id,
    req.user!.id
  );
  res.json({ data: board });
});

router.patch('/:id', validate(updateBoardSchema), async (req, res) => {
  const board = await boardService.updateBoard(
    req.params.id,
    req.user!.id,
    req.body
  );
  res.json({ data: board });
});

router.delete('/:id', async (req, res) => {
  await boardService.deleteBoard(req.params.id, req.user!.id);
  res.status(204).send();
});

export default router;
typescript
// packages/api/src/services/board.service.ts

import { prisma } from '../lib/prisma.js';
import { ForbiddenError, NotFoundError } from '../lib/errors.js';

export async function createBoard(data: { title: string; ownerId: string }) {
  return prisma.board.create({
    data: {
      title: data.title,
      ownerId: data.ownerId,
      members: {
        create: {
          userId: data.ownerId,
          role: 'ADMIN',
        },
      },
    },
    include: {
      members: { include: { user: { select: { id: true, name: true, email: true } } } },
    },
  });
}

export async function getBoardsForUser(userId: string) {
  return prisma.board.findMany({
    where: {
      members: { some: { userId } },
    },
    include: {
      _count: { select: { lists: true } },
      owner: { select: { id: true, name: true } },
    },
    orderBy: { updatedAt: 'desc' },
  });
}

export async function getBoardWithContents(boardId: string, userId: string) {
  const board = await prisma.board.findUnique({
    where: { id: boardId },
    include: {
      members: {
        include: {
          user: { select: { id: true, name: true, email: true, avatarUrl: true } },
        },
      },
      lists: {
        orderBy: { position: 'asc' },
        include: {
          cards: {
            orderBy: { position: 'asc' },
            include: {
              assignee: { select: { id: true, name: true, avatarUrl: true } },
            },
          },
        },
      },
    },
  });

  if (!board) throw new NotFoundError('Board not found');

  const isMember = board.members.some((m) => m.userId === userId);
  if (!isMember) throw new ForbiddenError('Not a member of this board');

  return board;
}

export async function updateBoard(
  boardId: string,
  userId: string,
  data: { title?: string }
) {
  await assertBoardMember(boardId, userId, ['ADMIN', 'MEMBER']);
  return prisma.board.update({
    where: { id: boardId },
    data,
  });
}

export async function deleteBoard(boardId: string, userId: string) {
  await assertBoardMember(boardId, userId, ['ADMIN']);
  await prisma.board.delete({ where: { id: boardId } });
}

async function assertBoardMember(
  boardId: string,
  userId: string,
  roles: string[]
) {
  const membership = await prisma.boardMember.findUnique({
    where: { boardId_userId: { boardId, userId } },
  });

  if (!membership) throw new ForbiddenError('Not a member of this board');
  if (!roles.includes(membership.role)) {
    throw new ForbiddenError(`Requires one of: ${roles.join(', ')}`);
  }
}

The pattern here is important: routes are thin, services have the logic. Claude follows this pattern because we specified it in CLAUDE.md. Without that specification, Claude might put the Prisma queries directly in the route handlers — functional but harder to test.

Now build the list and card endpoints following the same pattern.
Lists: POST, PATCH, DELETE, and PATCH for reordering.
Cards: full CRUD plus PATCH for moving between lists.

Phase 3: Frontend¶

Component Architecture¶

Create the React component structure for the board view. We need:
- BoardPage: fetches board data, handles WebSocket connection
- BoardHeader: title, members, settings
- ListView: column with cards, add card button, drag zone
- CardItem: card preview (title, labels, assignee avatar)
- CardModal: full card editor (opens on click)
- AddCard: inline form for creating new cards
Use React Query for data fetching and Zustand for drag state.

Claude generates the component tree. Here is the core board view:

tsx
// packages/web/src/components/Board/BoardPage.tsx

import { useParams } from 'react-router-dom';
import { useQuery } from '@tanstack/react-query';
import { api } from '../../lib/api';
import { BoardHeader } from './BoardHeader';
import { ListView } from './ListView';
import { AddList } from './AddList';
import { useWebSocket } from '../../hooks/useWebSocket';
import type { Board } from '@taskflow/shared';

export function BoardPage() {
  const { boardId } = useParams<{ boardId: string }>();

  const { data: board, isLoading, error } = useQuery({
    queryKey: ['board', boardId],
    queryFn: () => api.get<Board>(`/boards/${boardId}`),
    enabled: !!boardId,
  });

  useWebSocket(boardId);

  if (isLoading) return <BoardSkeleton />;
  if (error) return <ErrorState error={error} />;
  if (!board) return null;

  return (
    <div className="flex h-screen flex-col bg-blue-50">
      <BoardHeader board={board} />
      <div className="flex flex-1 gap-4 overflow-x-auto p-4">
        {board.lists.map((list) => (
          <ListView key={list.id} list={list} boardId={board.id} />
        ))}
        <AddList boardId={board.id} position={board.lists.length} />
      </div>
    </div>

// packages/api/src/ws/index.ts

import { WebSocketServer, WebSocket } from 'ws';
import { Server } from 'http';
import { verifyToken } from '../lib/jwt.js';
import { logger } from '../lib/logger.js';

interface Client {
  ws: WebSocket;
  userId: string;
  boardId: string | null;
}

const clients = new Map<WebSocket, Client>();

export function setupWebSocket(server: Server) {
  const wss = new WebSocketServer({ server, path: '/ws' });

  wss.on('connection', async (ws, req) => {
    const url = new URL(req.url!, `http://${req.headers.host}`);
    const token = url.searchParams.get('token');

    if (!token) {
      ws.close(4001, 'Missing token');
      return;
    }

    try {
      const payload = verifyToken(token);
      clients.set(ws, { ws, userId: payload.userId, boardId: null });

      ws.on('message', (data) => handleMessage(ws, data.toString()));
      ws.on('close', () => clients.delete(ws));

      ws.send(JSON.stringify({ type: 'connected' }));
    } catch {
      ws.close(4001, 'Invalid token');
    }
  });

  return { broadcast };
}

function handleMessage(ws: WebSocket, raw: string) {
  try {
    const msg = JSON.parse(raw);

    if (msg.type === 'join_board') {
      const client = clients.get(ws);
      if (client) {
        client.boardId = msg.boardId;
        logger.info({ userId: client.userId, boardId: msg.boardId }, 'Client joined board');
      }
    }

    if (msg.type === 'leave_board') {
      const client = clients.get(ws);
      if (client) {
        client.boardId = null;
      }
    }
  } catch (err) {
    logger.warn({ err }, 'Invalid WebSocket message');
  }
}

export function broadcast(boardId: string, event: object, excludeUserId?: string) {
  const message = JSON.stringify(event);
  for (const [, client] of clients) {
    if (
      client.boardId === boardId &&
      client.userId !== excludeUserId &&
      client.ws.readyState === WebSocket.OPEN
    ) {
      client.ws.send(message);
    }
  }
}

Frontend WebSocket hook with automatic reconnection:

typescript
// packages/web/src/hooks/useWebSocket.ts

import { useEffect, useRef, useCallback } from 'react';
import { useQueryClient } from '@tanstack/react-query';
import { useAuth } from './useAuth';

const WS_URL = import.meta.env.VITE_WS_URL || 'ws://localhost:3001/ws';
const RECONNECT_DELAY = 3000;
const MAX_RECONNECT_ATTEMPTS = 5;

export function useWebSocket(boardId: string | undefined) {
  const wsRef = useRef<WebSocket | null>(null);
  const reconnectAttempts = useRef(0);
  const queryClient = useQueryClient();
  const { token } = useAuth();

  const connect = useCallback(() => {
    if (!boardId || !token) return;

    const ws = new WebSocket(`${WS_URL}?token=${token}`);
    wsRef.current = ws;

    ws.onopen = () => {
      reconnectAttempts.current = 0;
      ws.send(JSON.stringify({ type: 'join_board', boardId }));
    };

    ws.onmessage = (event) => {
      const data = JSON.parse(event.data);

      switch (data.type) {
        case 'board_updated':
        case 'list_created':
        case 'list_updated':
        case 'list_deleted':
        case 'card_created':
        case 'card_updated':
        case 'card_moved':
        case 'card_deleted':
          queryClient.invalidateQueries({ queryKey: ['board', boardId] });
          break;
      }
    };

    ws.onclose = () => {
      if (reconnectAttempts.current < MAX_RECONNECT_ATTEMPTS) {
        reconnectAttempts.current++;
        setTimeout(connect, RECONNECT_DELAY);
      }
    };

    ws.onerror = () => ws.close();
  }, [boardId, token, queryClient]);

  useEffect(() => {
    connect();
    return () => {
      wsRef.current?.close();
    };
  }, [connect]);
}

Optimistic Updates¶

For a responsive feel, card movements update the UI immediately:

Add optimistic updates for card drag-and-drop. When the user drops a card:

2. Send the API request
3. If the request fails, roll back the UI and show an error toast

Claude integrates React Query’s optimistic update pattern:

typescript
// packages/web/src/hooks/useMoveCard.ts

import { useMutation, useQueryClient } from '@tanstack/react-query';
import { api } from '../lib/api';
import { toast } from '../lib/toast';
import type { Board } from '@taskflow/shared';

interface MoveCardInput {
  cardId: string;
  sourceListId: string;
  targetListId: string;
  position: number;
}

export function useMoveCard(boardId: string) {
  const queryClient = useQueryClient();

  return useMutation({
    mutationFn: (input: MoveCardInput) =>
      api.patch(`/boards/${boardId}/cards/${input.cardId}/move`, {
        targetListId: input.targetListId,
        position: input.position,
      }),

    onMutate: async (input) => {
      await queryClient.cancelQueries({ queryKey: ['board', boardId] });

      const previousBoard = queryClient.getQueryData<Board>(['board', boardId]);

      queryClient.setQueryData<Board>(['board', boardId], (old) => {
        if (!old) return old;

        const newLists = old.lists.map((list) => ({
          ...list,
          cards: [...list.cards],
        }));

        const sourceList = newLists.find((l) => l.id === input.sourceListId);
        const targetList = newLists.find((l) => l.id === input.targetListId);
        if (!sourceList || !targetList) return old;

        const cardIndex = sourceList.cards.findIndex((c) => c.id === input.cardId);
        if (cardIndex === -1) return old;

        const [card] = sourceList.cards.splice(cardIndex, 1);
        targetList.cards.splice(input.position, 0, card);

        return { ...old, lists: newLists };
      });

      return { previousBoard };
    },

    onError: (_err, _input, context) => {
      if (context?.previousBoard) {
        queryClient.setQueryData(['board', boardId], context.previousBoard);
      }
      toast.error('Failed to move card. Change has been reverted.');
    },

    onSettled: () => {
      queryClient.invalidateQueries({ queryKey: ['board', boardId] });
    },
  });
}

Lessons Learned¶

After building TaskFlow entirely with Claude Code, here are the key takeaways:

Use /plan for multi-step features, direct prompts for single-step tasks. The WebSocket feature needed a plan because it touched backend, frontend, and state management. Adding a new field to a card did not.

Work incrementally. Build the board CRUD first, then lists, then cards, then real-time, then tests. Each phase builds on the previous one and can be verified independently.

The CLAUDE.md pays for itself immediately. Without the “routes are thin, services have logic” directive, Claude would have produced working but inconsistent code. That one line saved dozens of corrections across 30+ files.

Let Claude write the tests. Integration tests are tedious. Claude writes them thoroughly and correctly. The key insight: specify what to test (happy paths, auth failures, validation errors), let Claude handle how.

Use /rewind when Claude goes off track. Twice during development, Claude made an architectural decision that diverged from the plan. Rather than trying to course-correct, /rewind to the last good state and re-prompt with more specific instructions.

Chapter Summary¶

  1. Start with a scaffold, then refine. One prompt generates the project structure. A few correction prompts shape it to your exact needs. This is faster than specifying everything upfront.

  2. Work incrementally, verify at each phase. Database, API, frontend, real-time, tests, deployment. Each phase is independently verifiable. Do not try to build everything in one prompt.

  3. The CLAUDE.md prevents architectural drift. Conventions documented in CLAUDE.md (thin routes, service layer, state management strategy) are followed consistently across dozens of files.

  4. Real-time features need /plan mode. Any feature that touches the entire stack benefits from a plan. Review the plan, then execute.

  5. Testing is where Claude Code saves the most time. Claude writes comprehensive tests — happy paths, error conditions, edge cases — faster than you can type them. Invest your time in specifying what to test.
